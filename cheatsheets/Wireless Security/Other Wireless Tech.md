# Другие беспроводные технологии

## Zigbee

**Что это:** низкопотребляющий, короткодействующий протокол для умных устройств (лампы, датчики, розетки). Работает на частотах 2.4 ГГц (глобально), 868 МГц (Европа), 915 МГц (США).

**Особенность:** mesh-топология — каждое устройство может ретранслировать данные соседям, расширяя сеть.

**Главная уязвимость:** при подключении нового устройства без предварительного ключа, координатор может передать сетевой ключ шифрования, защищенный только общеизвестным ключом по умолчанию. Злоумышленник, перехватив трафик в момент подключения, получает ключ и может расшифровать весь трафик.

**Пример атаки:** исследователи взломали Philips Hue — захватили контроль над лампочкой и установили вредоносное ПО на шлюз (CVE-2020-6007).

**Примеры устройств:** умные лампы, датчики движения, розетки, датчики открытия дверей.


## Z-Wave

**Что это:** протокол для домашней автоматизации (термостаты, жалюзи, замки, датчики). Работает на частоте 868 МГц (Европа), 908 МГц (США) — меньше помех, чем 2.4 ГГц. Более стандартизирован, чем Zigbee.

**Особенность:** большая дальность связи, чем у Zigbee. Mesh-топология.

**Уязвимости:**
- S0 (старый фреймворк безопасности) — ключ шифрования передавался со статическим значением "все нули", перехват тривиален.
- S2 (новый фреймворк) — использует Diffie-Hellman, но устройства могут быть принудительно понижены до S0, так как рукопожатие паринга не аутентифицировано.

**Пример атаки:** исследователи взломали умный замок Yale Conexis L1 (атака Z-Shave), извлекли сетевой ключ и получили постоянный доступ к замку.

**Примеры устройств:** умные замки, термостаты, датчики протечки, шторы.


## LoRaWAN (Long Range Wide Area Network)

**Что это:** протокол для дальней связи (до нескольких километров) с низким потреблением энергии. Работает на суб-гигагерцовых частотах (868 МГц в Европе, 915 МГц в США).

**Особенность:** дальность километры, но низкая скорость. Идеален для датчиков на открытой местности (поля, города).

**Уязвимость:**
- ABP (Activation by Personalization) — устройства используют статические ключи шифрования. Если счетчик фреймов сбрасывается (при переполнении или перезагрузке), злоумышленник может воспроизвести старые сообщения (replay attack).
- OTAA (Over-The-Air Activation) — безопаснее, генерирует свежие ключи при каждом подключении.

**Пример атаки:** replay-атака на LoRaWAN 1.0 — десинхронизация счетчиков между датчиком и сервером, блокировка легитимной связи.

**Примеры устройств:** датчики качества воздуха, мониторинг уровня воды в резервуарах, умное освещение улиц, сельскохозяйственные датчики.


## Сотовые технологии (LTE-M, NB-IoT)

**Что это:** передача данных через существующие мобильные сети (как смартфон). Работают в лицензируемом спектре операторов.

**Особенности:**
- LTE-M (LTE for Machines) — поддерживает роуминг, голос, скорость до 1 Мбит/с.
- NB-IoT (Narrowband IoT) — узкополосный, очень низкое потребление, до 100 кбит/с.
- Построены на 4G LTE, переходят на 5G.

**Уязвимости:**
- SIM-аутентификация и стандартное шифрование есть, но устройства часто находятся в удаленных местах → физический доступ и извлечение прошивки.
- Ограниченный софт, сложно патчить.
- Интерфейсы управления на шлюзах могут иметь дефолтные пароли и инъекции.

**Пример атаки:** исследователи Университета Пердью идентифицировали IoT-устройства в сети и эксплуатировали режимы низкого потребления для DoS и перехвата данных.

**Примеры устройств:** умные счетчики воды/электричества, трекеры автомобилей, промышленные датчики.


## Инфракрасный порт (Infrared / IR)

**Что это:** передача данных через световые сигналы, линия видимости (LOS), очень короткая дистанция. Частота около 38 кГц для модуляции.

**Уязвимости:**
- Нет аутентификации и шифрования.
- Атакующий может записать сигнал пульта ДУ и воспроизвести его (replay attack).
- Команды следуют стандартным протоколам (NEC, RC5, Sony SIRC) — можно создать команды без захвата сигнала, просто зная спецификацию.

**Пример атаки:** Raspberry Pi с IR-трансивером за 5$ управляет телевизорами, кондиционерами, проекторами с другого конца комнаты.

**Примеры устройств:** пульты ДУ телевизоров, кондиционеров, проекторов, аудиосистем.


## Сравнение технологий (без таблиц)

**Zigbee и Z-Wave** — похожи (локальная автоматизация, mesh). Разница: Z-Wave (868/908 МГц) менее загружен, лучше проходит стены, более стандартизирован. Zigbee (2.4 ГГц) дешевле, глобальная совместимость, но больше помех.

**LoRaWAN** — для дальних расстояний (километры) и очень низкого потребления. Не подходит для видео или голоса.

**LTE-M / NB-IoT** — используют готовую мобильную инфраструктуру (вышки операторов). Есть абонентская плата. NB-IoT внутри помещений проникает лучше, чем LTE-M.

**Infrared** — умирающая технология, но ещё встречается. Безопасность отсутствует полностью.


## Защита (для каждой технологии)

**Zigbee:**
- Использовать install codes — уникальный ключ на устройство, чтобы сетевой ключ не передавался под общеизвестным ключом.
- Включать шифрование для всех узлов.

**Z-Wave:**
- Отключать обратную совместимость с S0 (только S2).
- Если S0 нужен — требовать явного подтверждения пользователя при подключении.

**LoRaWAN:**
- Использовать OTAA вместо ABP (свежие ключи при каждом подключении).
- Если ABP неизбежен — регулярная ротация ключей и мониторинг счетчиков фреймов.

**Cellular IoT:**
- Менять дефолтные пароли на шлюзах.
- Размещать устройства за VPN или частным APN.
- Регулярно обновлять прошивки.

**Infrared:**
- Деактивировать IR-приемники на сетевых устройствах (если не используются).
- Не размещать IR-управляемые устройства в зоне прямой видимости из окна/двери.

**Общая рекомендация для всех:**
- Сегментировать IoT/беспроводные устройства в отдельные сети/VLAN.
- Изолировать их от систем с чувствительными данными.


## Для пентестера (инструменты и подходы)

**Zigbee:**
- KillerBee, Zigator, Attify Zigbee Tool
- Устройства: CC2531 USB dongle (с прошивкой Zigbee sniffer)

**Z-Wave:**
- Z-Force (исследовательский)
- PC-based sniffers требуют специального оборудования

**LoRaWAN:**
- RTL-SDR + plugins (для пассивного сниффинга)
- LoRa Bridge, LoRaWAN sniffer
- Устройства: HackRF One, LimeSDR

**Infrared:**
- Raspberry Pi + IR-трансивер (VS1838B, TSOP382)
- LIRC (Linux Infrared Remote Control)
- Или просто записать сигнал на телефон с ИК-портом

**Общий подход:**
1. Определить протокол (спектроанализатор, SDR)
2. Перехватить парунг/активацию (самый уязвимый момент)
3. Извлечь ключи или воспроизвести захваченные команды
4. Атаковать шлюз (он обычно на Wi-Fi/Ethernet, имеет веб-интерфейс)


## Ключевые выводы

- **Zigbee и Z-Wave** уязвимы в момент подключения новых устройств.
- **LoRaWAN** с ABP статичен и подвержен replay-атакам.
- **Сотовый IoT** — физический доступ к устройству раскрывает многое.
- **Infrared** вообще не имеет защиты (no auth, no encryption).
- Сегментация сети — самый эффективный способ защиты.
- Пентестеру нужны SDR, специализированные даглы и терпение.
