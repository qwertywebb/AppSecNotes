# Bluetooth Security

## Что такое Bluetooth

Технология беспроводной связи между устройствами через прямое сопряжение (pairing). Работает на частоте 2,4 ГГц, предназначена для короткой дистанции.

**Примеры устройств:** беспроводные клавиатуры и мыши, наушники и колонки, смарт-часы, автомобильные системы, IoT и умные устройства.


## Две основные разновидности Bluetooth

**Classic Bluetooth (классический)** — решает тяжелые задачи: потоковое аудио, передача файлов. Требует больше энергии, поддерживает постоянное соединение.

**Bluetooth Low Energy (BLE, энергоэффективный)** — для устройств, которые отправляют небольшие порции данных эпизодически: фитнес-трекеры, умные замки, датчики. Потребляет мало энергии.

**Простая аналогия:** Classic Bluetooth — это телефонный звонок (постоянное соединение). BLE — это SMS (быстро и легко).


## Различия в безопасности

**Classic Bluetooth** — проходит полноценный процесс сопряжения с согласованием общего ключа. В современных версиях используется Secure Simple Pairing (SSP) с криптографическим обменом ключами.

**BLE** — поддерживает разные методы сопряжения. Самый распространенный в дешевых устройствах — "Just Works" (без проверки пользователя). Это позволяет злоумышленнику вклиниться между устройствами и перехватывать данные. BLE версий 4.2 и выше поддерживают LE Secure Connections. Версии 4.0 и 4.1 используют устаревший LE Legacy Pairing с более слабой защитой.

**Исторический:** старые версии Bluetooth использовали простой PIN-код (до 2.1). Коды по умолчанию (0000, 1234) легко угадывались, процесс сопряжения можно было перехватить или перебрать атакой.


## Как происходит сопряжение (Pairing)

**Для Classic Bluetooth (SSP):**
1. Устройства обнаруживают друг друга
2. Выбирается метод сопряжения (например, сравнение чисел на экранах)
3. Происходит обмен криптографическими ключами
4. Устанавливается защищенное соединение

**Для BLE (LE Secure Connections):**
1. Устройства обнаруживают друг друга
2. Выбирается метод сопряжения (Just Works, Passkey Entry, Numeric Comparison, OOB)
3. Происходит обмен ключами
4. Устанавливается защищенное соединение

**Примечание:** внешний вид сопряжения зависит от устройства. Что-то просит подтвердить одинаковые числа, что-то использует NFC, а простые устройства подключаются автоматически.


## Основные риски безопасности

**Необязательная обнаруживаемость (Discoverable Mode)** — устройство в режиме обнаружения видно окружающим. Атакующий может его найти и попытаться атаковать.

**Слабые механизмы сопряжения** — старые устройства с простыми PIN-кодами уязвимы для брутфорса.

**Неавторизованное сопряжение** — атакующий может попытаться подключиться к устройству, если контроль подтверждения слабый или отключен.

**Bluejacking** — отправка нежелательных сообщений на Bluetooth-устройства.

**Bluesnarfing** — попытка доступа к данным через Bluetooth-соединение (контакты, сообщения, файлы).

**Bluebugging** — эксплуатация уязвимости для получения контроля над функциями устройства (звонки, SMS, интернет).

**Отсутствие обновлений** — устаревшие реализации Bluetooth содержат известные уязвимости.


## Зона поражения

Атакующий должен находиться в непосредственной близости от устройства (обычно до 10-30 метров). Места, где часто проводятся Bluetooth-атаки: офисы, конференции, аэропорты, кафе.


## Меры защиты (по типам устройств)

**Для всех Bluetooth устройств:**
- Выключать Bluetooth, когда не используется
- Не оставлять устройства в режиме обнаружения
- Регулярно обновлять прошивки

**Для Classic Bluetooth:**
- Использовать Secure Simple Pairing (Bluetooth 2.1+)
- Не использовать PIN-коды по умолчанию (0000, 1234)

**Для BLE:**
- Использовать LE Secure Connections (Bluetooth 4.2+)
- Избегать устройств с методом "Just Works" для чувствительных данных
- Внедрять проверку пользователя при сопряжении

**Для организаций:**
- Политика использования Bluetooth на корпоративных устройствах
- Обучение сотрудников рискам Bluejacking и Bluesnarfing
- Мониторинг необычной Bluetooth-активности в зоне офиса


## Ключевые выводы

- Bluetooth имеет ограниченную дальность, но это не делает его безопасным по умолчанию
- BLE с методом "Just Works" не обеспечивает проверку пользователя
- Устаревшие версии Bluetooth (до 2.1) и BLE Legacy Pairing (4.0-4.1) имеют слабую защиту
- Bluejacking — неопасен, Bluesnarfing и Bluebugging — опасны
- Основная защита: выключать Bluetooth, обновлять устройства, использовать современные версии

**Ментальная модель:** уязвимое Bluetooth-устройство — это приоткрытое окно на первом этаже. Кажется, что ничего страшного, но этого достаточно для проникновения. Правильная конфигурация и осведомленность решают всё.
