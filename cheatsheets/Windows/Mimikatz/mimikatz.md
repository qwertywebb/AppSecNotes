**Mimikatz** – это мощный инструмент для извлечения паролей и других данных аутентификации из памяти Windows. Простыми словами, если хакер (или специалист по безопасности) получает доступ к компьютеру под правами администратора, он может запустить Mimikatz и «вытащить» из оперативной памяти пароли, хеши и билеты Kerberos других пользователей, которые заходили на эту машину .

Давайте разберем его подробнее.

### 🎯 Основная цель Mimikatz

Это инструмент для **пост-эксплуатации (post-exploitation)**. Это означает, что он используется уже **после** того, как система была скомпрометирована и злоумышленник получил права администратора . Его главное назначение:
*   Красть данные аутентификации (логины и пароли) из памяти скомпрометированной машины .
*   Использовать эти данные для **латерального перемещения (lateral movement)** – то есть для перехода на другие компьютеры в той же сети .

### ⚙️ Как это работает и что можно украсть?

Mimikatz взаимодействует с критически важным процессом Windows — **Local Security Authority Subsystem Service (LSASS)**. LSASS отвечает за аутентификацию пользователей и, для удобства, хранит данные о недавних входах в своей памяти . Mimikatz просто обращается к этой памяти и извлекает оттуда ценную информацию .

Вот список того, что Mimikatz может украсть:

*   **Пароли в открытом виде (Plaintext Passwords):** Если в системе включен определенный протокол, Mimikatz может достать пароль прямо в читаемом виде .
*   **NTLM хеши (NTLM hashes):** Это зашифрованная версия пароля. Но даже хеш очень ценен, так как его можно использовать для атаки **Pass-the-Hash** .
*   **Kerberos билеты (Kerberos Tickets):** Билеты используются для аутентификации в сетях с доменом (Active Directory). Их кража позволяет провести атаку **Pass-the-Ticket** .

### 💥 Популярные атаки с использованием Mimikatz

Украденные данные используются для нескольких классических атак:

*   **Pass-the-Hash (PtH):** Злоумышленник использует украденный хеш пароля (а не сам пароль) для авторизации на другом компьютере в сети . Это один из самых популярных способов перемещения по сети.
*   **Pass-the-Ticket (PtT):** Аналогично, для доступа к другим системам используется украденный Kerberos-билет .
*   **Golden Ticket:** Это «король» атак. Атакующий, используя Mimikatz, может создать поддельный, «золотой» Kerberos-билет, который дает ему **неограниченный доступ к любому ресурсу в домене** .
*   **Overpass-the-Hash (или Pass-the-Key):** Этот метод использует хеш для получения полного Kerberos-билета, что позволяет обходить некоторые виды защиты .

### 🛡️ Почему это опасно?

Mimikatz стал стандартным инструментом и для «красных команд» (red teams), и для реальных киберпреступников . Его используют для:
*   **Эскалации привилегий:** С его помощью можно получить права администратора домена, начав с обычного пользователя .
*   **Распространения по сети:** Захватив одну машину, атакующий крадет с нее пароли и переходит на следующую, заражение распространяется как «вирус» .
*   **Закрепления в сети:** Создание «золотых билетов» позволяет сохранять доступ годами, даже если первоначальная уязвимость будет закрыта .

Из-за своей опасности Mimikatz является **«hacktool»** и детектируется большинством антивирусов, которые рассматривают его как потенциально опасное ПО .

### 📋 Основные команды (шпаргалка)


Вот список самых ходовых команд Mimikatz 

---

**Повысить привилегии до Debug** — `privilege::debug`

**Повысить привилегии до уровня системы** — `token::elevate`

**Извлечь хеши паролей локальных пользователей из SAM** — `lsadump::sam`извлекает хеши паролей локальных пользователей из базы данных SAM (Security Account Manager) Windows.

SAM — это файл в системе Windows (C:\Windows\System32\config\SAM), где хранятся хеши паролей всех локальных пользователей. Доступ к нему есть только у системы и процессов с правами SYSTEM.

**Вытащить пароли и хеши из памяти (LSASS)** — `sekurlsa::logonpasswords`

**Атака Pass-the-Hash (подключение по хешу)** — `sekurlsa::pth /user:USER /domain:DOMAIN /ntlm:HASH`

**Вытащить Kerberos-билеты из текущей сессии** — `kerberos::list`

**Атака Pass-the-Ticket (использовать украденный билет)** — `kerberos::ptt <путь_к_файлу_билета>`

**Создать "Золотой билет" (Golden Ticket) для неограниченного доступа в домене** — `kerberos::golden /user:USER /domain:DOMAIN /sid:SID /krbtgt:HASH /ptt`
