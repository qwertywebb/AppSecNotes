# Port Forwarding With Socat — Cheatsheet

## Что такое Socat

Socat — утилита для перенаправления портов, альтернатива SSH, когда SSH недоступен.

**Минусы:** нужно загружать socat на промежуточный хост (больше следов).
**Плюсы:** проще в использовании, не требует SSH.


## Базовая команда

```bash
socat TCP4-LISTEN:ЛОКАЛЬНЫЙ_ПОРТ,fork TCP4:ЦЕЛЕВОЙ_IP:ЦЕЛЕВОЙ_ПОРТ
```

**Параметры:**
- `TCP4-LISTEN:порт` — слушать порт на текущей машине
- `fork` — обрабатывать несколько подключений (без него socat закроется после первого)
- `TCP4:IP:порт` — куда перенаправлять трафик


## Сценарий 1: Проброс порта с промежуточного хоста на цель

**Ситуация:** У тебя есть доступ к PC-1, но нет доступа к серверу. PC-1 может подключиться к серверу.

**Задача:** Сделать так, чтобы ты мог подключиться к серверу через PC-1.

### Команда на PC-1

socat TCP4-LISTEN:3389,fork TCP4:3.3.3.3:3389

**Что происходит:**
- PC-1 открывает порт 3389
- Ты подключаешься к PC-1:3389
- Socat перенаправляет твой трафик на сервер 3.3.3.3:3389

### Разрешить подключение в фаерволе Windows (если нужно)
netsh advfirewall firewall add rule name="Open Port 3389" dir=in action=allow protocol=TCP localport=3389

### Подключение с Kali
xfreerdp /v:PC-1_IP:3389 /u:user /p:pass


## Сценарий 2: Проброс порта с промежуточного хоста на атакующую машину (обратный)

**Ситуация:** Сервер должен подключиться к Kali (например, для reverse shell). PC-1 выступает как ретранслятор.

### Команда на PC-1
socat TCP4-LISTEN:80,fork TCP4:1.1.1.1:80

**Что происходит:**
- PC-1 открывает порт 80
- Сервер подключается к PC-1:80
- Socat перенаправляет трафик на Kali (1.1.1.1:80)


## Когда использовать Socat

- SSH недоступен или заблокирован
- Нужна простая пересылка портов без аутентификации
- Есть возможность загрузить файл на промежуточный хост


## Быстрая шпаргалка

# Проброс с промежуточной на цель (входящий)
socat TCP4-LISTEN:ЛОКАЛЬНЫЙ_ПОРТ,fork TCP4:ЦЕЛЬ:ПОРТ

# Проброс с промежуточной на Kali (исходящий)
socat TCP4-LISTEN:ПОРТ,fork TCP4:KALI_IP:ПОРТ

# Разрешить порт в фаерволе Windows
netsh advfirewall firewall add rule name="Название" dir=in action=allow protocol=TCP localport=ПОРТ
