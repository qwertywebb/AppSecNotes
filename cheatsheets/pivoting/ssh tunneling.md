# SSH Port Forwarding

## Одна важная вещь

**`-L` и `-R` делают почти одно и то же: открывают порт на твоей Kali.**

Разница только в том, **кто инициирует соединение с промежуточным хостом**.


## `-L` (Local) — обычный случай

Ты подключаешься к промежуточному хосту (ты клиент, он сервер).

```bash
ssh jdoe@192.168.13.71 -L 13389:192.168.13.100:3389 -N
```

**Что происходит:** На Kali открывается порт 13389. Трафик идет через промежуточный хост к цели.


## `-R` (Remote) — обратный случай

Промежуточный хост подключается к тебе (он клиент, ты сервер).

# На Kali (должен быть запущен SSH сервер)
sudo systemctl start ssh

# На промежуточном хосте (PC-1)
ssh kali_user@10.0.0.1 -R 3389:192.168.13.100:3389 -N

**Что происходит:** Тоже открывается порт на Kali. Трафик идет через промежуточный хост к цели.

## Когда что использовать

**`-L`** — промежуточный хост доступен для подключения (обычный случай).

**`-R`** — промежуточный хост за NAT или файрволом, ты не можешь подключиться к нему напрямую, но он может подключиться к тебе.


## Простая запоминалка

- **`-L`** = Local = ты подключаешься к хосту (Local connection)
- **`-R`** = Remote = хост подключается к тебе (Remote connection)

Результат одинаковый: порт на Kali ведет к цели.


## Пример

# Если можешь подключиться к промежуточному хосту
ssh jdoe@192.168.13.71 -L 13389:192.168.13.100:3389 -N

# Если не можешь подключиться, но промежуточный хост может подключиться к тебе
# На Kali: sudo systemctl start ssh
# На промежуточном хосте: ssh kali@10.0.0.1 -R 3389:192.168.13.100:3389 -N

В обоих случаях сервис будет доступен локально 127.0.0.1:13389
