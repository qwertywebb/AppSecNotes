# АТАКА: KERBEROASTING

## Что это:
Запрос TGS (Service Ticket) для учетных записей с SPN и взлом этого билета офлайн.

## Почему это работает:
- Любой аутентифицированный пользователь может запросить TGS для любого SPN
- TGS зашифрован хэшем пароля сервисной учетки
- Если пароль слабый — взламывается офлайн
- Сервисные учетки часто имеют слабые пароли и много прав

## Кто цель:
- Учетки с зарегистрированными SPN (службы, приложения)
- Пример: svc_printer, sql_service, http_service

## Как найти SPN в домене:
GetUserSPNs.py домен/пользователь:пароль -dc-ip IP_DC -request

## Команда для запроса билетов:
GetUserSPNs.py thm.loc/claire:'Password123!' -dc-ip 192.168.11.100 -request

## Формат билета для взлома:
$krb5tgs$23$*svc_printer$THM.LOC$... (начинается с $krb5tgs$)

## Взлом Hashcat:
hashcat -m 13100 service_ticket.txt /usr/share/wordlists/rockyou.txt

## Режимы Hashcat для Kerberos:
- 13100: Kerberos 5 TGS-REP (RC4)
- 19600: Kerberos 5 TGS-REP (AES)

## Что делать с результатом:
- Использовать пароль для аутентификации
- Часто сервисные учетки имеют права на несколько серверов

## Защита:
- Длинные пароли для сервисных учеток (30+ символов)
- Групповые управляемые сервисные учетки (gMSA)
- Регулярная смена паролей
- Мониторинг аномальных запросов TGS (много от одного пользователя)
