# АТАКА: ВЗЛОМ NTLM ХЭША

## Почему это работает:
- NTLM хэши хранятся в AD без соли (unsalted)
- Используется алгоритм MD4 (быстрый)
- Одинаковые пароли → одинаковые хэши
- Современное железо (GPU) может перебирать миллиарды хэшей в секунду

## Когда атакуют:
- Получили NTLM хэш из дампа SAM, ntds.dit или памяти lsass
- Хэш не удалось использовать напрямую (Pass-the-Hash не работает)
- Нужен именно пароль (например, для доступа к сервисам без NTLM)

## Инструменты:
- Hashcat (режим 1000 для NTLM)
- John the Ripper

## Пример команды:
hashcat -m 1000 hash.txt /usr/share/wordlists/rockyou.txt

## Формат хэша для Hashcat:
939B0058BC6DD834ABC4CC08CFEFEA69

## Расшифровка формата NTLM хэша:
phillip:1106:aad3b435b51404eeaad3b435b51404ee:939B0058BC6DD834ABC4CC08CFEFEA69:::
                    |                             |
             LM хэш (пустой)               NTLM хэш (то что нужно)

## Что делать с результатом:
- Использовать пароль для аутентификации
- smbclient.py "домен/пользователь:пароль@IP"

## Защита:
- Длинные сложные пароли (>14 символов)
- Использовать фразы, а не слова из словаря
- Отключить NTLM где возможно
