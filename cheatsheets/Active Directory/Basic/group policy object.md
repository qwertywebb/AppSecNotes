# GROUP POLICY OBJECTS (GPO)

GPO — это набор настроек (политик), который применяется к OU. GPO позволяют централизованно управлять конфигурациями пользователей и компьютеров.

GPO могут содержать настройки для:
- Computer Configuration (применяются к компьютерам)
- User Configuration (применяются к пользователям)

# Как работают GPO:

1. Создаешь GPO в папке Group Policy Objects
2. Линкуешь (связываешь) GPO с нужным OU
3. GPO применяется к целевому OU и ко всем вложенным OU (наследование)
4. По умолчанию GPO применяется ко всем Authenticated Users

# Распространение GPO:

GPO хранятся в сетевой папке SYSVOL на Domain Controller.
Путь по умолчанию: C:\Windows\SYSVOL\sysvol\

Компьютеры синхронизируют GPO каждые ~2 часа.
Принудительная синхронизация на конкретной машине:
gpupdate /force

# Пентест-нюансы про GPO:

Атакующий ищет в SYSVOL файлы Groups.xml, Services.xml, ScheduledTasks.xml, Drives.xml, DataSources.xml — там могут быть пароли в шифрованном виде, ключ известен и можно расшифровать, будет получен доступ к учетной записи, которая и создала эту GPO.

Если атакующий имеет права на запись в SYSVOL или редактирование GPO — он может внедрить вредоносную политику (например, добавить задачу в планировщик на всех компьютерах домена). Это называется GPO Abuse.

Пример атаки: атакующий создает GPO, которая добавляет локального администратора на все рабочие станции, или запускает вредоносный скрипт при входе пользователя.

# GPO CHEATSHEET ДЛЯ ПЕНТЕСТЕРА

Где найти GPO: Group Policy Management → домен → Group Policy Objects

Как посмотреть, какие GPO применены к компьютеру:
gpresult /r

Как принудительно применить GPO:
gpupdate /force

Где лежат GPO на DC: C:\Windows\SYSVOL\sysvol\[домен]\Policies\

Что искать в SYSVOL как атакующий:
- *.xml файлы с паролями (cpassword)
- Скрипты входа (logon scripts) с паролями внутри
- Scheduled Tasks с учетными данными

Опасные права для атакующего:
- Write на GPO (может изменить существующую политику)
- Create GPO (может создать свою)
- Link GPO на OU (может применить вредоносную политику)

Признак плохого администрирования (уязвимость):
- Лежат старые GPO с cpassword
- Любой пользователь может читать SYSVOL
- Права на редактирование GPO у не-админов

# GPP (Group Policy Preferences) уязвимость

В старых GPO (до Windows 2012) пароли в Groups.xml, Services.xml, ScheduledTasks.xml хранились в зашифрованном виде с известным ключом AES.

Расшифровка: gpp-decrypt <cpassword>
Или онлайн: https://cpassword.com/

Microsoft выпустила патч KB2962486, но на старых системах это до сих пор работает.

# Security Filtering в GPO

По умолчанию GPO применяется ко всем Authenticated Users.

Чтобы исключить группу:
- Удали Authenticated Users из фильтрации
- Добавь только нужные группы с разрешением Allow
- (Deny используется редко, но он имеет приоритет)

Проверить, кто получит GPO: вкладка Scope → Security Filtering

# Ключевые правила работы с GPO (для запоминания):

1. GPO наследуются от родительского OU к дочерним.
2. Computer Configuration применяется когда компьютер включается/обновляет политики. Линкуй на OU с компьютерами.
3. User Configuration применяется когда пользователь логинится. Линкуй на OU с пользователями.
4. Security Filtering: Authenticated Users получают все. Чтобы исключить группу — удали Authenticated Users и добавь нужные.
5. Уязвимость GPP: старые GPO (Groups.xml) содержат шифрованные пароли с известным ключом.
