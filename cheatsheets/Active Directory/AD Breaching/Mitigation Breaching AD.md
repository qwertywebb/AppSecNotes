# ЗАЩИТА ОТ BREACHING AD (что искать атакующему)

## Secrets Management (защита от утечки секретов)

Что защищает:
- Хранилища секретов (HashiCorp Vault, Azure Key Vault)
- Pre-commit хуки (TruffleHog, Gitleaks) — сканируют перед коммитом
- Ротация паролей после утечки

Что искать атакующему:
- Репозитории без этих защит
- История коммитов не проверяется
- Секреты все еще в коде

## Password Policies (защита от password spraying)

Что защищает:
- Длина пароля 14+ символов
- Запрет популярных паролей и паттернов (company+year, season+year)
- Уникальные пароли для новых аккаунтов
- Lockout threshold: 5-10 попыток за 30 минут

Что искать атакующему:
- Lockout threshold высокий (>10) — можно больше попыток
- Политика паролей слабая (минимальная длина 8)
- Нет запрета на "Company2025!"

## Device Hardening (защита от LDAP Passback)

Что защищает:
- Смена дефолтных паролей на устройствах
- LDAPS (порт 636) вместо LDAP (389)
- Ограничение доступа к админке по IP/VLAN
- Низкопривилегированные учетки для устройств

Что искать атакующему:
- Дефолтные пароли (admin:admin)
- LDAP без шифрования (порт 389)
- Админка доступна из общей сети

## File Share Security (защита от file-based coercion)

Что защищает:
- Минимальные права на запись (least privilege)
- Мониторинг файлов .url, .lnk, .scf
- Аудит доступа к шарам

Что искать атакующему:
- Шары с доступом на запись для многих пользователей
- Нет мониторинга подозрительных файлов

## NTLM Hardening (защита от захвата NTLM хэшей)

Что защищает:
- Отключение NTLMv1 (только NTLMv2)
- SMB Signing (защита от relay)
- Блокировка исходящего SMB трафика (порт 445)
- Миграция на Kerberos

Что искать атакующему:
- NTLMv1 включен (старые системы)
- SMB Signing отключен (можно relay)
- Исходящий SMB не блокируется

## Network Segmentation (общая защита)

Что защищает:
- Управляющие интерфейсы в отдельных VLAN
- Ограничение доступа к Jenkins, Git, другим сервисам
- MFA для критических сервисов

Что искать атакующему:
- Jenkins/Git доступны из общей сети
- Нет MFA даже для важных систем
- Сегментация не настроена

# ЧТО ЗАЩИЩАЕТ ОТ КАКОЙ АТАКИ (быстрая справка)

┌─────────────────────────┬────────────────────────────────────┐
│ Атака                    │ Защита                             │
├─────────────────────────┼────────────────────────────────────┤
│ Credential Discovery    │ Secrets vault, pre-commit hooks    │
│ (Git, Jenkins)          │ ротация паролей                    │
├─────────────────────────┼────────────────────────────────────┤
│ Password Spraying       │ Длина 14+, запрет популярных       │
│                         │ паролей, lockout 5-10/30min        │
├─────────────────────────┼────────────────────────────────────┤
│ LDAP Passback           │ Смена дефолтных паролей, LDAPS     │
│                         │ изолированные VLAN                 │
├─────────────────────────┼────────────────────────────────────┤
│ File-based Coercion     │ Минимальные права на запись,       │
│                         │ мониторинг .url файлов             │
├─────────────────────────┼────────────────────────────────────┤
│ NTLM Capture            │ Отключение NTLMv1, SMB Signing,    │
│                         │ блокировка порта 445 на выход      │
└─────────────────────────┴────────────────────────────────────┘

SMB Signing — защищает от NTLM Relay:
- Подписывает каждый SMB пакет
- Атакующий не может изменить или ретранслировать подписанный пакет
- Без SMB Signing — relay возможен
