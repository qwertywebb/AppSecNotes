# АУТЕНТИФИКАЦИЯ VS АВТОРИЗАЦИЯ (ВАЖНОЕ РАЗЛИЧИЕ)

Аутентификация — доказательство личности. Ответ на вопрос "Ты тот, за кого себя выдаешь?"
Авторизация — определение прав. Ответ на вопрос "Что тебе разрешено делать?"

Аутентификация ВСЕГДА происходит первой. Сначала докажи кто ты, потом смотрим что тебе можно.

# ЧТО МОЖЕТ БЫТЬ КРЕДАМИ (не только пароль)

1. Username + Password — самый частый вариант
2. Certificates — криптографический сертификат (например, смарт-карта)
3. Hashes — хэши паролей (не предназначены для передачи, но используются в атаках)

# КЛЮЧЕВОЙ НЮАНС (для пентеста)

Другие протоколы (LDAP, HTTP, SMB) сами по себе НЕ выполняют аутентификацию.
Они полагаются на Kerberos или NetNTLM.

Пример: когда ты подключаешься к SMB шаре (\\server\share), 
аутентификацию обрабатывает Kerberos или NetNTLM, а не сам SMB.

Это важно, потому что:
- Атаки на Kerberos (Golden Ticket, Silver Ticket) работают независимо от протокола
- Перехваченный NetNTLM хэш можно использовать против SMB, HTTP, LDAP

# NTLM Relay — атака на NetNTLM

Перехваченный NetNTLM ответ (response) можно переиспользовать против:
- SMB
- HTTP (IIS, Exchange, SharePoint)
- LDAP
- MSSQL

Нельзя использовать против сервиса, который ТРЕБУЕТ только Kerberos и отключил NTLM.
Но по умолчанию NTLM включен почти везде.

# Ключевое понимание для пентестера

Любая аутентификация в AD сводится к Kerberos или NetNTLM.
HTTP, SMB, LDAP — это просто протоколы поверх. Они не отвечают за аутентификацию.

Если понять Kerberos и NetNTLM — атаки станут прозрачными:
- Pass-the-Hash → работа с NTLM
- Pass-the-Ticket → работа с Kerberos
- Golden/Silver Ticket → подделка Kerberos билетов
- NTLM Relay → манипуляция NetNTLM
- Kerberoasting → запрос TGS и взлом пароля сервиса офлайн

# ТРЕБОВАНИЯ К ПРАВАМ ДЛЯ АТАК

Атаки, доступные ЛЮБОМУ доменному пользователю:
- Kerberoasting (запрос TGS для SPN)
- AS-REP Roasting (если отключена pre-auth)
- Перечисление пользователей/групп

Атаки, требующие хэш/билет (полученный другими способами):
- Pass-the-Hash
- Pass-the-Ticket

Атаки, требующие права на Domain Admin или доступ к DC:
- Golden Ticket (нужен хэш KRBTGT)
- DCSync (права Replicating Directory Changes)
- Дамп ntds.dit
