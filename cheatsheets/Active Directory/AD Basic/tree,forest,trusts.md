# МНОЖЕСТВЕННЫЕ ДОМЕНЫ: TREES, FORESTS, TRUSTS

Когда компания растет, одного домена становится мало. Например: филиалы в разных странах, разные законы, независимые IT-команды.

## Tree (Дерево)

Tree — это несколько доменов с ОБЩИМ пространством имен (namespace).

Пример: корневой домен thm.local, дочерние домены uk.thm.local и us.thm.local.

Особенности Tree:
- У каждого дочернего домена свой Domain Controller
- Domain Admin из uk.thm.local не управляет us.thm.local
- Политики настраиваются независимо

## Forest (Лес)

Forest — это несколько деревьев с РАЗНЫМИ пространствами имен.

Пример: компания THM купила MHT Inc. Появились домены thm.local и mht.local в одном лесу.

Forest — это максимальная граница AD. Все что внутри леса — может иметь доверие.

## Группы в лесу:

- Domain Admins — управляют своим доменом
- Enterprise Admins — управляют ВСЕМИ доменами в лесу (важно для пентеста!)

## Trust Relationships (Доверительные отношения)

Trust — это связь между доменами, позволяющая пользователю из домена А получить доступ к ресурсам в домене Б.

### One-way trust (одностороннее доверие):

Если домен AAA доверяет домену BBB (AAA → BBB), то пользователь из BBB может получить доступ к ресурсам в AAA.

ВНИМАНИЕ: направление доверия противоположно направлению доступа.

Пример: thm.local доверяет mht.local. Пользователь из mht.local заходит на ресурс в thm.local.

### Two-way trust (двустороннее доверие):

Оба домена доверяют друг другу. Пользователи из любого домена могут получить доступ к ресурсам другого.

По умолчанию внутри Tree и Forest создаются двухсторонние транзитивные доверия.

### Важно (для пентеста):

Доверие между доменами НЕ ДАЕТ автоматически доступ ко всем ресурсам. Оно только РАЗРЕШАЕТ авторизацию. Администратор должен еще выдать конкретные права на папки, принтеры и т.д.

Но для пентеста: доверие — это путь. Если скомпрометирован один домен, можно двигаться через доверие в другой.


# ПЕНТЕСТ TREES, FORESTS, TRUSTS

1. Enterprise Admins — главная цель в лесу. Кто владеет Enterprise Admins — владеет всеми доменами леса.

2. Атака через доверие (Trust Hopping):
   - Скомпрометировал домен A
   - Смотришь с кем у него доверительные отношения
   - Двигаешься в домен B, если доверие двустороннее или направлено в нужную сторону

3. SID History — опасный атрибут при миграции пользователей между доменами. Если атакующий получил контроль над объектом с SID History от Enterprise Admins — он может получить права в другом домене.

4. Golden Ticket работает в пределах одного домена. Для движения через лес нужен Forest Golden Ticket (DCSync на Enterprise Admins).

5. При атаке на forest часто используется доверие: можно запросить TGS для сервиса в другом домене, имея TGT из скомпрометированного домена (атака TGT delegation).

### ВАЖНОЕ:
# Движение между доменами через доверие (пентест)

Внутри дерева (thm.local → uk.thm.local, us.thm.local):
- Дочерние домены НЕ доверяют друг другу напрямую
- Путь: child1 → root → child2

Golden Ticket действует ТОЛЬКО в рамках одного домена.
Для движения между доменами нужны:
- Межлесное доверие (cross-forest trust)
- SID History атаки
- Атаки на службы с делегированием (Resource-based Constrained Delegation)

# Атака из дочернего домена на родительский

Если Enterprise Admin логинился в дочерний домен, его SID может быть в SID History.
Зная krbtgt дочернего домена, можно создать Golden Ticket с SID родителя.
Результат: ты становишься Enterprise Admin в родительском домене.

Команда в Mimikatz для такого билета:
kerberos::golden /user:Administrator /domain:child.thm.local /krbtgt:<хэш> /sid:<SID_родительского_домена> /sidadmin:<SID_Enterprise_Admin>
