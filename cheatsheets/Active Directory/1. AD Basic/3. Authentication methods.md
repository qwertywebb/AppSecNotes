# ПРОТОКОЛЫ АУТЕНТИФИКАЦИИ В WINDOWS DOMAIN

Когда пользователь хочет получить доступ к ресурсу в домене (например, к сетевой папке или принтеру), сервис не проверяет пароль сам. Он отправляет запрос к Domain Controller.

В Windows доменах используются два протокола:

1. Kerberos — современный, используется по умолчанию в новых версиях Windows
2. NetNTLM — старый, оставлен для совместимости

## KERBEROS (простое объяснение)

Kerberos использует систему "билетов". Один раз прошел аутентификацию — получил билет. С этим билетом можешь ходить к разным сервисам без повторного ввода пароля.

### Участники процесса:

- Клиент (пользователь, который хочет получить доступ)
- Сервис (например, файловый сервер, к которому хочет подключиться клиент)
- KDC (Key Distribution Center) — служба на Domain Controller, которая выдает билеты

### Этапы работы Kerberos:

**Этап 1 — Получение TGT (Ticket Granting Ticket):**
- Клиент отправляет на KDC свой username и timestamp, зашифрованный хэшем его пароля
- KDC проверяет и выдает TGT (билет на получение других билетов)
- TGT зашифрован хэшем пароля учетной записи krbtgt
- Вместе с TGT клиент получает Session Key

**Этап 2 — Получение TGS (Ticket Granting Service) для конкретного сервиса:**
- Клиент хочет подключиться к файловому серверу. Он отправляет на KDC свой TGT + SPN (имя сервиса, например cifs/fileserver01)
- KDC проверяет TGT (может расшифровать, потому что знает пароль krbtgt)
- KDC выдает TGS — билет для доступа к конкретному сервису
- TGS зашифрован хэшем пароля того аккаунта, под которым запущен сервис (например, паролем компьютера fileserver01$)

**Этап 3 — Подключение к сервису:**
- Клиент отправляет TGS на файловый сервер
- Сервер расшифровывает TGS своим паролем, проверяет и пускает клиента

### Ключевые термины Kerberos (ЗАПОМНИТЬ ДЛЯ ПЕНТЕСТА):

- TGT — билет на получение билетов. Позволяет запрашивать доступ к любым сервисам, на которые у пользователя есть права.
- TGS — билет на конкретный сервис. Позволяет подключиться только к одному сервису.
- krbtgt — учетная запись в AD, чьим хэшом зашифрован TGT. Если атакующий украдет хэш krbtgt, он может создать поддельный TGT для любого пользователя (атака Golden Ticket).
- SPN (Service Principal Name) — уникальный идентификатор сервиса в домене. Формат: service_type/host_name (пример: cifs/dc01.thm.local)
- Session Key — временный ключ для шифрования общения между клиентом и KDC/сервисом


## NetNTLM

Старый протокол. Работает через "запрос-ответ" (challenge-response). Пароль не передается по сети.

### Этапы работы NetNTLM:

1. Клиент говорит серверу: "я хочу подключиться"
2. Сервер отправляет клиенту случайное число (challenge)
3. Клиент комбинирует хэш своего пароля с этим числом, создает ответ (response) и отправляет серверу
4. Сервер пересылает challenge и response на Domain Controller
5. DC сам вычисляет, какой должен быть response (потому что у DC есть хэш пароля пользователя), и сравнивает
6. Если совпало — доступ разрешен

### Важное отличие от Kerberos:

- Kerberos: получил билет один раз, используешь его многократно
- NetNTLM: каждое подключение требует нового challenge-response

# Различие Kerberos и NetNTLM (кто проверяет)

- NetNTLM: Сервис отправляет credentials на DC. DC проверяет. Сервис НЕ МОЖЕТ проверить сам.
- Kerberos: DC выдает TGT/TGS. Сервис может сам расшифровать TGS (своим паролем) и проверить клиента без обращения к DC.

### Пентест-нюанс про NetNTLM:

Атакующий может перехватить NetNTLM-трафик и использовать его для атаки "Pass-the-Hash" или ретрансляции (NTLM Relay) на другой сервер.

Пример: ты перехватил NetNTLM-ответ пользователя. Не зная его пароль, ты можешь отправить этот ответ на другой сервер и представиться этим пользователем.

----

Билеты Kerberos имеют время жизни (по умолчанию 10 часов).
TGT и TGS кешируются на клиенте.
Это один из минусов Kerberos для пентестера: украденный билет можно использовать пока не истек срок.

---- 

Golden Ticket — поддельный TGT, созданный с помощью хэша krbtgt.
Дает атакующему доступ к ЛЮБОМУ ресурсу в домене.
Лечится только сменой пароля krbtgt ДВАЖДЫ.

----

Почему Kerberos лучше NetNTLM:
- Нет передачи хэша или его производных
- Есть взаимная аутентификация (клиент тоже проверяет сервер)
- Защита от replay-атак через timestamp

# КЛЮЧЕВЫЕ МОМЕНТЫ KERBEROS ДЛЯ ПЕНТЕСТА

1. TGT зашифрован хэшом krbtgt.
2. TGS зашифрован хэшом сервиса (например, компьютера).
3. Украденный TGT/TGS можно использовать до истечения срока (обычно 10 часов).
4. Golden Ticket — поддельный TGT, созданный с хэшом krbtgt. Дает доступ ко всему домену.
5. TGS можно подделать для конкретного сервиса, зная хэш этого сервиса — это Silver Ticket.
