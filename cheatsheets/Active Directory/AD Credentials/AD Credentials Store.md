# CREDENTIAL STORES — ГДЕ WINDOWS ХРАНИТ ПАРОЛИ

## Зачем это знать пентестеру

Windows и AD хранят учетные данные в разных местах для разных целей.
Каждое место требует своего подхода к извлечению и дает разные типы данных.

---

# 1. LSASS MEMORY — ПАМЯТЬ ПРОЦЕССА LSASS

**Что это:** Local Security Authority Subsystem Service — процесс, который управляет аутентификацией.

**Что хранит (в памяти):**
- NTLM и LM хэши паролей
- Kerberos билеты (TGT и TGS)
- Иногда plaintext пароли

**Зачем это нужно Windows:** Поддержка Single Sign-On (SSO) — пользователь ввел пароль один раз, а система использует его для доступа к разным сервисам.

**Как получить (пентест):**
- SYSTEM доступ к машине
- Дамп памяти процесса lsass.exe

**Инструменты:** Mimikatz (sekurlsa::logonpasswords, sekurlsa::minidump)

---

# 2. SAM + SYSTEM HIVES — ЛОКАЛЬНЫЕ УЧЕТНЫЕ ЗАПИСИ

**Что это:** SAM (Security Accounts Manager) — реестровая база данных с хэшами локальных пользователей.

**Что хранит:**
- Хэши паролей локальных аккаунтов (включая локального администратора)

**Зачем это нужно Windows:** Аутентификация при локальном входе на компьютер (когда компьютер не в домене или вход локальной учеткой).

**Нюанс:** Хэши зашифрованы. Для расшифровки нужен BootKey из SYSTEM hive.

**Где физически:**
- %SystemRoot%\system32\config\SAM
- %SystemRoot%\system32\config\SYSTEM

**Как получить (пентест):**
- SYSTEM доступ
- Экспорт через реестр или Volume Shadow Copy

**Инструменты:** Mimikatz (lsadump::sam), secretsdump.py

---

# 3. LSA SECRETS — СЕКРЕТЫ LSA

**Что это:** Хранилище секретов в реестре (HKLM\SECURITY\Policy\Secrets)

**Что хранит:**
- Кэшированные доменные учетные данные (для офлайн входа)
- Пароли в plaintext от scheduled tasks и служб
- Пароли RDP сессий

**Зачем это нужно Windows:** 
- Позволяет залогиниться на ноутбуке без связи с DC (кэш)
- Хранит пароли для автоматических задач

**Как получить (пентест):**
- SYSTEM или Admin доступ
- Через LSARPC интерфейс

**Инструменты:** secretsdump.py (с локальными админ правами)

---

# 4. DPAPI VAULT — ХРАНИЛИЩЕ ПРИЛОЖЕНИЙ

**Что это:** Data Protection API — криптографическая служба Windows для защиты секретов приложений.

**Что хранит:**
- Сохраненные пароли от приложений (браузеры, WiFi, RDP)
- Каждый пользователь имеет свой master key

**Зачем это нужно Windows:** Приложения могут безопасно хранить пароли без необходимости спрашивать их каждый раз.

**Нюанс:** Master key зашифрован паролем пользователя. Чтобы расшифровать все секреты пользователя, нужны его пароль и мастер-ключ.

**Где физически:** %APPDATA%\Microsoft\Protect

**Как получить (пентест):**
- Доступ к файлам пользователя
- Знание его пароля (или дамп LSASS)

**Инструменты:** Mimikatz (vault::list, vault::cred /export)

---

# 5. NTDS.DIT — БАЗА ДАННЫХ AD (ТОЛЬКО НА DC)

**Что это:** Core Active Directory database — главная база домена.

**Что хранит:**
- ВСЕ учетные записи домена
- NTLM хэши паролей ВСЕХ пользователей
- Kerberos ключи

**Зачем это нужно Windows:** Центральное хранилище для аутентификации всего домена.

**Почему это главная цель пентестера:** Кто владеет NTDS.dit — владеет всем доменом.

**Как получить (пентест):**
- Доступ к DC
- Репликация через MS-DRSR (DCSync)
- Или прямой доступ к файлу

**Инструменты:** secretsdump.py -just-dc, Mimikatz (lsadump::dcsync)

---

# СВОДКА

**LSASS Memory**
- Что получаем: NTLM хэши, Kerberos билеты, иногда plaintext
- Инструменты: mimikatz (sekurlsa::logonpasswords)

**SAM + SYSTEM**
- Что получаем: Хэши локальных пользователей
- Инструменты: mimikatz (lsadump::sam)

**LSA Secrets**
- Что получаем: Кэшированные креды, пароли служб
- Инструменты: secretsdump.py

**DPAPI Vault**
- Что получаем: Пароли из браузеров, RDP, WiFi
- Инструменты: mimikatz (vault::*)

**NTDS.dit (DC only)**
- Что получаем: ВСЕ хэши домена
- Инструменты: secretsdump.py -just-dc

---

# КЛЮЧЕВОЕ ПОНИМАНИЕ

1. LSASS — живая память (сессии, которые активны прямо сейчас)
2. SAM — локальные учетки (не доменные)
3. LSA Secrets — кэш для офлайн входа и пароли задач
4. DPAPI — пароли из приложений (браузеры, RDP)
5. NTDS.dit — СВЯТОЙ ГРААЛЬ (все хэши домена)

Для пентестера: начинай с LSASS (быстро), затем SAM (если нужно локальных), затем NTDS.dit (если есть доступ к DC).


# DCSYNC — КАК ПОЛУЧИТЬ NTDS.DIT БЕЗ ДОСТУПА К ФАЙЛУ

DCSync имитирует поведение контроллера домена и запрашивает репликацию базы AD у другого DC.

Для атаки нужны права:
- Replicating Directory Changes (DS-Replication-Get-Changes)
- Replicating Directory Changes All (DS-Replication-Get-Changes-All)

Обычно эти права есть у Domain Admins, Enterprise Admins, Administrators.

Команды:
# Impacket
secretsdump.py -just-dc DOMAIN/username@DC_IP

# Mimikatz (на DC или с правами)
lsadump::dcsync /user:krbtgt /domain:DOMAIN
