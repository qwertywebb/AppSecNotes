# USER ENUMERATION — ПЕРЕЧИСЛЕНИЕ ПОЛЬЗОВАТЕЛЕЙ

## Зачем это нужно

Перед password spraying нужно знать, какие пользователи существуют в домене.
Можно получить список через разные протоколы, даже без учетки.

# 1. LDAP ANONYMOUS BIND

## Что это

Некоторые LDAP серверы разрешают анонимные запросы (без пароля).
Можно вычитать пользователей, группы, компьютеры.

## Проверка анонимного доступа

ldapsearch -x -H ldap://TARGET_IP -s base

-x — простая аутентификация (анонимная)
-H — сервер LDAP
-s base — только базовая информация

## Запрос всех пользователей

ldapsearch -x -H ldap://TARGET_IP -b "dc=tryhackme,dc=loc" "(objectClass=person)"

-b — база для поиска (base DN)


# 2. ENUM4LINUX-NG

## Что это

Автоматизирует enumeration через SMB/RPC:
- Пользователи и группы
- Шары
- Password policy
- OS информация

## Команда (всё сразу)

enum4linux-ng -A TARGET_IP -oA results

-A — все функции (all)
-oA — вывести в файлы (YAML, JSON)


# 3. RPC NULL SESSION

## Что это

Через SMB (порт 445) и IPC$ можно подключиться с пустыми учетками.
RPC (Remote Procedure Call) дает много информации о домене.

## Подключение через rpcclient

rpcclient -U "" TARGET_IP -N

-U "" — пустой пользователь
-N — без пароля

## Полезные команды внутри rpcclient

enumdomusers — список всех пользователей
enumalsgroups builtin — локальные группы
enumalsgroups domain — доменные группы
enumdomains — список доменов
queryuser RID — информация о пользователе по RID
lsaenumsid — перечисление SID

## RID Cycling (ручной перебор)

Если enumdomusers не работает, перебираем RID вручную:

for i in $(seq 500 2000); do 
  echo "queryuser $i" | rpcclient -U "" -N TARGET_IP 2>/dev/null | grep -i "User Name"
done

500-2000 — диапазон RID
500 — Administrator
501 — Guest
512-514 — группы (Domain Admins, Domain Users, Domain Guests)
1000+ — обычные пользователи


# 4. KERBRUTE (USER ENUMERATION ЧЕРЕZ KERBEROS)

## Что это

Отправляет AS-REQ запросы к KDC (порт 88).
Если пользователь существует → ответ KRB5KDC_ERR_PREAUTH_REQUIRED
Если не существует → KRB5KDC_ERR_C_PRINCIPAL_UNKNOWN

## Запуск

kerbrute userenum --dc TARGET_IP -d DOMAIN users.txt

--dc — IP контроллера домена
-d — домен
-o — сохранить результат (опционально)

## Почему kerbrute полезен после других инструментов

enum4linux и rpcclient могут вернуть:
- Отключенные учетки
- Не доменные учетки
- Фейковые (honeypot)

Kerbrute проверяет только активные AD учетки.


# Сводка по методам user enumeration

Метод | Без учетки? | Протокол | Что дает
------|-------------|----------|---------
LDAP Anonymous | ✅ Да | LDAP (389) | Пользователи, группы
enum4linux-ng | ✅ Да | SMB/RPC | Всё: юзеры, группы, шары, политика
rpcclient null | ✅ Да | SMB/RPC | Пользователи через RID
kerbrute | ✅ Да | Kerberos (88) | Только активные AD учетки
