# AD ENUMERATION — ПЕРЕЧИСЛЕНИЕ ДОМЕНА

## Цель этапа

У нас есть доступ к сети (VPN), но нет учетных данных. Нужно:
1. Найти все живые хосты в сети
2. Определить, какой из них Domain Controller
3. Узнать имя домена
4. Выявить открытые сервисы для дальнейших атак


# HOST DISCOVERY (поиск живых хостов)

## fping — быстрый ICMP сканер

Особенность: не ждет ответа от каждого хоста, переходит к следующему.

fping -agq 10.211.11.0/24

Параметры:
-a — показать живые хосты
-g — генерировать список из CIDR
-q — тихий режим (без мусора)

Пример вывода:
10.211.11.1
10.211.11.10
10.211.11.20
10.211.11.250

## Nmap ping scan
nmap -sn 10.211.11.0/24

-sn — только ping, без сканирования портов


# PORT SCANNING (поиск сервисов)

## Ключевые порты AD для обнаружения DC

Расшифровка портов:
- 88 — Kerberos (билеты, аутентификация)
- 135 — RPC (удаленные вызовы)
- 139 — NetBIOS (старый SMB)
- 389 — LDAP (справочник AD)
- 445 — SMB (файловые шары)
- 636 — LDAPS (безопасный LDAP)
- 464 - kpasswd (смена паролей kerberos)

## Целевой scan на эти порты

nmap -p 88,135,139,389,445 -sV -sC -iL hosts.txt

-sV — определить версии сервисов
-sC — NSE скрипты (default)
-iL — читать список IP из файла

## Как узнать DC

DC имеет открытыми:
- 88 (Kerberos)
- 389 (LDAP)
- 445 (SMB)

В выводе nmap будет видно:
- "Windows Server ..."
- Имя домена в баннере

## Полный scan портов (если нужно)
nmap -sS -p- -T3 -iL hosts.txt -oN full_port_scan.txt

-sS — SYN scan (тихий)
-p- — все порты (1-65535)
-T3 — нормальная скорость
-oN — вывод в файл
