# Windows Internals — Processes (Конспект)

## Что такое процесс

**Процесс** — это экземпляр выполняемой программы. Каждое приложение в Windows запускает как минимум один процесс. Процесс предоставляет ресурсы для выполнения программы: память, код, дескрипторы, контекст безопасности и потоки.

Процесс — это не сама программа, а «контейнер», в котором программа выполняется.

---

## Компоненты процесса

| Компонент | Описание |
|-----------|----------|
| **Private Virtual Address Space** | Виртуальное адресное пространство, выделенное процессу |
| **Executable Program** | Код и данные, загруженные в адресное пространство |
| **Open Handles** | Ссылки на системные ресурсы (файлы, реестр, объекты) |
| **Security Context** | Токен доступа: пользователь, группы, привилегии |
| **Process ID (PID)** | Уникальный числовой идентификатор процесса |
| **Threads** | Потоки выполнения внутри процесса |

---

## Структура процесса в памяти

| Компонент | Описание |
|-----------|----------|
| **Code** | Исполняемый код |
| **Global Variables** | Глобальные переменные |
| **Process Heap** | Куча для динамического выделения памяти |
| **Process Resources** | Дополнительные ресурсы процесса |
| **Environment Block** | Структура с информацией о процессе |

---

## Примеры стандартных процессов Windows

| Процесс | Назначение |
|---------|------------|
| `MsMpEng` | Microsoft Defender (антивирус) |
| `wininit` | Обработка клавиатуры и мыши |
| `lsass` | Хранение учётных данных (Local Security Authority Subsystem Service) |

---

## Почему процессы интересны для пентестера

Процессы часто используются для сокрытия вредоносной активности. Основные техники:

| Техника | Описание |
|---------|----------|
| **Process Injection** | Внедрение кода в легитимный процесс |
| **Process Hollowing** | Замена кода в существующем процессе |
| **Process Masquerading** | Маскировка вредоносного процесса под легитимный |

---

## Как наблюдать за процессами

| Инструмент | Назначение |
|------------|------------|
| **Task Manager** | Встроенный диспетчер задач |
| **Process Explorer** | Расширенный просмотр процессов (Sysinternals) |
| **Process Hacker 2** | Альтернативный просмотрщик процессов |
| **Procmon** | Мониторинг процессов и реестра (Sysinternals) |

---

## Основная информация о процессе (Task Manager)

| Поле | Описание | Пример |
|------|----------|--------|
| **Name** | Имя процесса (обычно от приложения) | `conhost.exe` |
| **PID** | Уникальный идентификатор | `7408` |
| **Status** | Состояние (Running, Suspended) | `Running` |
| **User name** | Пользователь, запустивший процесс | `SYSTEM` |


## Ключевые выводы

- Процесс — это контейнер для выполнения программы.
- Каждый процесс имеет уникальный PID.
- Процессы могут быть атакованы (инъекция, халлоуинг, маскировка).
- Для наблюдения за процессами используй Task Manager или Process Explorer.


## Шпаргалка

```cmd
# Просмотр процессов в командной строке
tasklist

# Информация о конкретном процессе
tasklist /fi "PID eq 1234"

# Завершить процесс по PID
taskkill /PID 1234 /F

# Завершить процесс по имени
taskkill /IM notepad.exe /F
```
