## RSA — это алгоритм шифрования, основанный на сложности факторизации больших чисел.
### Если у тебя есть два простых числа p и q, их легко перемножить, но обратно — разложить произведение на множители — очень трудно.

# Простыми словами.

# RSA - алгоритм шифрования, основанный на сложности факторизации больших чисел. Берем 2 простых числа, перемножаем их, потом проивзодим еще несколько обратимых операций и получаем пару ключей (публичный и закрытый). Дальше если вернутся к произведению чисел, очень сложно расшифровать множители, если числа например с 300 нулями.


from sympy import factorint
from Crypto.Util.number import inverse, long_to_bytes

# Дано
n = 43941819371451617899582143885098799360907134939870946637129466519309346255747
e = 65537
c = 9002431156311360251224219512084136121048022631163334079215596223698721862766

# 1. Факторизация n (если n маленький или слабый)
factors = factorint(n)
p, q = list(factors.keys())
ИЛИ ИЩЕМ P и Q через https://factordb.com
print(f"p = {p}")
print(f"q = {q}")

# 2. Вычисляем φ(n)
phi_n = (p - 1) * (q - 1)

# 3. Вычисляем d (приватную экспоненту)
d = inverse(e, phi_n)

# 4. Расшифровываем
plaintext_int = pow(c, d, n)
flag = long_to_bytes(plaintext_int)
print(f"Decrypted: {flag.decode()}")


# Когда RSA ломается:
* p и q близки	Ферма (sqrt(n) ± ...)
* Общий множитель с другим n	gcd(n1, n2)
* Маленький e (3,5,7)	Атака Хастеда / CRT
* Нет паддинга	Математическая атака
* Маленький d	Атака Винера
