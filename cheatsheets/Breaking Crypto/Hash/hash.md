# Хэширование это односторонний процесс по трансформации входного ввода в строку с фиксированным размером,  который нельзя обратить.

### Уязвимые алгоритмы хэширования:
- MD5 - у двух разных входов может быть одинаковый хэш (коллизия). Атакующий делает такой payload, чтобы хш совпадал у нужным хэшем.
- SHA1 - также уязвим к коллизиям, но сложнее найти их, чем в MD5.

- SHA256 - считается безопасным, но он оптимизирован по скорости, этим может воспользоваться атакующий, чтобы перебрать большое количество входов(атакой грубой силы). На мощной gpu миллион можно перебрать миллион хэшей в секунду. Но отлично подходит для поверка контрольных сумм, целостности данных.

# Для хранения паролей Argon2, bcrypt, and PBKDF2 т.к их сделали специально с учетом больших вычислительных затрат, что затрудняет атаки грубой силы.

# HMAC (Hash-based Message Authentication Code) - механизм для проверки целостности сообщений. Используется хэш-функция и секретный ключ. Если у атакующего есть доступ к секретному ключу, то он может проверить целостность и манипулировать сообщениями.
