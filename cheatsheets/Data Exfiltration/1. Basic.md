# Data Exfiltration — Вводный конспект

## Что такое Data Exfiltration

**Data Exfiltration** — это процесс несанкционированного копирования и перемещения чувствительных данных из внутренней сети организации наружу.

Важно понимать: это **пост-компрометационная** активность. Атакующий уже получил доступ к сети, провёл разведку, нашёл ценные данные и теперь выносит их.

В модели **Cyber Kill Chain** это происходит на последнем этапе — **Actions on Objectives**.


## Зачем это делают

- Кража учётных данных (пароли, токены, ключи)
- Хищение банковской информации
- Вынос бизнес-стратегий и планов
- Получение исходного кода проектов
- Сбор персональных данных сотрудников

Также exfiltration используется для:
- Скрытия вредоносной активности
- Обхода средств защиты (DLP, NGFW)


## Три основных сценария использования

### 1. Традиционный вынос данных (Exfiltrate)

Данные перемещаются из внутренней сети наружу. Атакующему не важен ответ от сервера — трафик идёт **в одну сторону** (внутри → снаружи). Данные сохраняются на сервере атакующего.

### 2. Командно-контрольная связь (C2 Communications)

Атакующий отправляет команды на заражённую машину, получает результаты. Трафик идёт **в обе стороны** (внутрь и наружу). Используются нестандартные протоколы.

### 3. Туннелирование (Tunneling)

Через скомпрометированный хост создаётся канал, позволяющий атакующему получить доступ ко всей внутренней сети. Трафик идёт **постоянно в обе стороны**.


## Какие техники будут рассмотрены

- Exfil через TCP socket + Base64
- Exfil через DNS
- Exfil через HTTPS (POST-запрос)
- Exfil через ICMP
