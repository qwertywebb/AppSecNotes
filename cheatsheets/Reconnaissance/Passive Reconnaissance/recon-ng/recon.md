# Recon-ng - это фреймворк отлично подходит для автоматизации рутинных задач OSINT, особенно на начальном этапе, когда нужно собрать максимум информации за короткое время.

# В отличие от универсальных скриптов, Recon-ng имеет модульную структуру (как Metasploit), встроенную базу данных и готовые интеграции с десятками онлайн-сервисов . Это позволяет не тратить время на написание собственных парсеров и настройку API.

### 🎯 Основные команды

**Управление рабочим пространством**
```bash
recon-ng                     # Запуск фреймворка
recon-ng -w my_workspace     # Создать и войти в новую рабочую область
workspaces list              # Показать все области 
workspaces create project_x  # Создать новую
workspaces load project_x    # Загрузить существующую
workspaces remove project_x  # Удалить
```

**Управление модулями**
```bash
marketplace search whois                     # Найти модули по ключевому слову 
marketplace install <module_path>            # Установить модуль 
marketplace install all                      # Установить все модули
modules load <module_path>                   # Загрузить модуль для использования
back                                         # Выгрузить модуль (выйти из него) 
```

**Работа с модулем**
```bash
show options                 # Показать параметры текущего модуля 
set <параметр> <значение>    # Установить параметр (например, SOURCE facebook.com) 
unset <параметр>             # Сбросить параметр
run                          # Запустить модуль после настройки 
```

**Работа с базой данных и выводом**
```bash
show domains                 # Показать собранные домены 
show hosts                   # Показать найденные хосты 
query select * from hosts    # Выполнить произвольный SQL-запрос к базе 
reporting/csv                # Экспорт в CSV (модуль) 
reporting/html               # Экспорт в HTML 
```

### 🔑 Управление ключами API

Многие модули требуют ключи API для отправки запросов (например, Google, Bing, Shodan) . Они хранятся отдельно:
```bash
keys list                    # Список всех сохраненных ключей
keys add <name> <api_key>    # Добавить ключ (например, keys add shodan_api YOUR_KEY) 
keys delete <name>           # Удалить ключ
```
