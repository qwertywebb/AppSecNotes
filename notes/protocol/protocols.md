# PROTOCOLS CHEATSHEET — Что такое протокол и зачем он нужен


## Что такое протокол

Протокол — это набор правил, по которым два устройства (клиент и сервер) обмениваются данными. В контексте веба протокол определяет, как именно браузер должен обработать ссылку: открыть ли страницу, скачать файл, выполнить код или запустить приложение.

Протокол указывается в начале URL перед двоеточием, например в `https://example.com` протокол — `https`.


## Основные протоколы

| Протокол | Описание |
|----------|----------|
| **http://** | Обычный HTTP-запрос к серверу. Есть заголовок `Origin: домен`. |
| **https://** | То же, но с шифрованием (TLS/SSL). Origin присутствует. |
| **file://** | Локальный файл на компьютере пользователя. Origin: null. |
| **data://** | Данные встроены прямо в URL. Без запроса к серверу. Origin: null. |
| **ftp://** | Передача файлов по протоколу FTP. |
| **ws://** | WebSocket — постоянное соединение (незашифрованное). |
| **wss://** | WebSocket Secure — постоянное соединение (зашифрованное). |
| **mailto://** | Открывает почтовый клиент для отправки письма. |
| **tel://** | Набирает телефонный номер. |
| **javascript://** | Выполняет JavaScript-код прямо в адресной строке (в атрибуте `href`). |


## DATA: Protocol — встраивание данных в URL

### Формат

```
data:[MIME-тип][;charset][;base64],<данные>
```

### Примеры

```
data:text/plain,Hello
data:text/html,<h1>Hi</h1>
data:text/html;base64,PGgxPkhpPC9oMT4=
data:image/png;base64,iVBORw0...
data:application/json,{"key":"value"}
```

### Что это даёт

- Можно передать изображение, HTML, JSON или другой контент без запроса к серверу
- Полезно для внедрения вредоносного кода (XSS) в уязвимых приложениях
- Не вызывает сетевого запроса, поэтому не виден в логах сервера

## Origin Rules — откуда пришёл запрос

Origin — это домен, с которого был отправлен запрос. Браузер использует Origin для политики CORS и безопасности.

| Протокол | Origin |
|----------|--------|
| `http://site.com` | `http://site.com` |
| `https://site.com` | `https://site.com` |
| `file:///path` | `null` |
| `data:...` | `null` |
| `about:blank` | `null` |
| `javascript:...` | Наследует Origin родителя |

**Если Origin = null**, то запрос считается «недоверенным». CORS не сработает, куки не будут отправлены, сессии недоступны.


## Как это использовать в пентесте

### 1. Обход CORS через `data:` или `file:`

Некоторые приложения проверяют только `Origin` заголовок. Если они разрешают `Origin: null`, ты можешь использовать `data:` или `file:` для отправки запросов.

### 2. XSS через `javascript:`

```html
<a href="javascript:alert(document.cookie)">click</a>
```

Если приложение позволяет вставлять ссылки, это может привести к XSS.

### 3. Обход валидации URL через `data:`

Если приложение проверяет `http://` или `https://`, но не проверяет `data:`, можно внедрить вредоносный контент.

### 4. SSRF через `file://`

Если приложение загружает контент по URL, попробуй использовать `file:///etc/passwd` для чтения локальных файлов.


## Примеры уязвимостей

| Протокол | Уязвимость | Пример |
|----------|------------|--------|
| `javascript:` | XSS | `<a href="javascript:alert(1)">` |
| `data:` | XSS / SSRF | `<img src="data:text/html,<script>alert(1)</script>">` |
| `file:` | LFI (SSRF) | `file:///etc/passwd` |
| `http://localhost` | SSRF | Доступ к внутренним сервисам |

## Шпаргалка (быстрые команды)

```bash
# Проверить, как сервер обрабатывает file://
curl -v "http://target.com/fetch?url=file:///etc/passwd"

# Проверить data: URI
curl -v "http://target.com/fetch?url=data:text/plain,Hello"

# Проверить javascript: в ссылке
curl -v "http://target.com/comment?text=<a href=\"javascript:alert(1)\">Click</a>"

# Отправить data: как изображение
curl -v "http://target.com/upload?img=data:image/png;base64,iVBORw0..."
```


## Защита

- Ограничить список разрешённых протоколов (белый список)
- Не разрешать `javascript:`, `data:`, `file:` в пользовательских ссылках
- Валидировать все URL с помощью `parse_url` (PHP) или `URL` (Python)
- Настроить CSP для блокировки `data:` и `javascript:`
- Не доверять `Origin: null` — блокировать или требовать дополнительную проверку
